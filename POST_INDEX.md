# 포스트 색인 (중복 방지용)

신규 주제 작성 전 이 파일을 먼저 확인한다.  
**같은 핵심 논점**을 다루는 포스트는 신규 작성 금지.  
포스트 추가·수정·삭제 시 이 파일도 함께 갱신.

---

## 관리 규칙

1. **동일 논점 중복 작성 금지**
   - 같은 각도로 다루는 포스트는 하나만 존재
   - 기존 포스트를 리뉴얼하려면: 기존 파일을 아카이브 후 교체

2. **비슷하지만 다른 각도는 허용**
   - 소니 PS 전략 실패 분석 ≠ 소니의 이상한 하드웨어 결정 (범위 다름)
   - MFC→WinUI3 마이그레이션 = MFC 사용 안 할 수 있는가? (파이 겹침 → 금지)

3. **겹침 판단 기준**
   - "두 포스트의 결론이 같은 질문에 답하는가?" → 겹침
   - "독자가 한 포스트만 읽어도 다른 포스트가 불필요해지는가?" → 겹침

## 중복 체크 빠른 가이드

새 주제를 쓰기 전, 아래 질문에 답한다.

```
1. 관련 카테고리 테이블에서 비슷한 파일 있는가?
   → 있다면 "핵심 논점" 컬럼 확인

2. 두 포스트가 답하는 질문이 같은가?
   → Yes → 금지. 기존 포스트 수정 또는 아카이브 후 교체

3. 독자가 한 포스트만 읽으면 다른 포스트가 불필요해지는가?
   → Yes → 금지

4. 범위가 겹치지만 각도가 다른가?
   예: sony-playstation-strategy (경영 실패 구조) ≠ 소니 하드웨어 실패 사례 (제품 결정)
   → OK
```
