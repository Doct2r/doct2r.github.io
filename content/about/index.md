## 이 블로그에 대해

개발과 기술에 관한 내용을 정리하는 개인 기술 블로그입니다.

특정 기술 하나를 매뉴얼처럼 설명하기보다, "이게 왜 이렇게 생겼는가"와 "이 둘은 어디서부터 다른가"를 따라가는 글을 주로 씁니다. AI IDE들이 왜 죄다 VS Code 포크인지, 프레임워크에 패턴을 강제로 얹으면 왜 불행해지는지, GUI는 정말 사용자를 위한 것인지, 새 언어의 홍보 문구는 실제 탄생 이유와 같은지 — 결과물보다 그 결과가 나온 맥락과 경계를 더 파고드는 편입니다.

주로 다루는 주제:

- **프로그래밍 언어** — C의 자식들(C++·Objective-C·Java·JS·Go 등 문법 계보), 베이직의 흥망, TypeScript·Rust·코틀린·C#의 탄생 배경, 새 언어의 셀링 포인트와 실제 동기의 간극, 내 언어를 직접 만들어 쓴다는 것의 현실
- **개발 문화·도구** — 프레임워크의 암묵적 한계(MFC와 MVC 사례), 협업이 1+1을 못 넘는 구조적 원인, 카나리/베타/알파 같은 릴리스 용어의 두 축, CLI가 지금도 쓰이는 이유와 GUI가 정말 사용자를 위한 것인지, 개발자라면 알아야 할 엔지니어들
- **애플/맥 개발** — AppKit부터 SwiftUI까지 UI 툴킷 계보, 맥 개발 시 확인해야 할 체크리스트
- **경영·산업 구조** — 오라클의 소송 중심 사업 모델, 표준과 독점의 차이, 대기업-하청 수익구조, 경쟁국 기술이전의 실제 사례
- **하드웨어** — Intel과 AMD의 설계 철학 차이, 녹투아가 성능의 아이콘이 된 이유
- **AI** — LLM 시대에 새 프로그래밍 언어가 필요한지, AI·ML 용어 정리, AI 근거의 신뢰 문제

같은 주제라도 이미 다른 각도로 다룬 적이 있으면 새로 쓰지 않고, 다루는 각도가 명확히 다를 때만 새 글을 올립니다. 전체 목록은 [글 목록](/archive/)에서 볼 수 있습니다.
