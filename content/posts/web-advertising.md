지금 이 페이지를 여는 그 짧은 순간에도, 어딘가에서는 당신이라는 사람에게 광고 하나를 보여줄 권리를 놓고 실시간 경매가 벌어지고 있다. 이 경매는 100밀리초 안에 끝난다. 웹 광고는 어쩌다 "사람이 지면을 파는 것"에서 "기계가 사람을 사고파는 것"으로 바뀌었을까. 기원부터 지금의 구조까지 따라가 본다.

---

## 1. 기원 — 광고는 원래 대화로 팔렸다

웹이 등장하기 전 광고는 **관계 기반 협상**으로 팔렸다. 신문사 영업 담당이 광고주에게 전화를 걸어 지면 위치와 크기를 협상하고, 계약서에 사인하면 몇 주 뒤 그 지면에 광고가 실렸다. TV·라디오도 마찬가지로 방송사 영업팀과 광고 대행사가 직접 협상했다.

```
전통 광고 거래:
  광고주 → 광고 대행사 → 매체사 영업팀 → (협상) → 계약 → 집행
  거래 단위: 개별 계약. 가격은 사람이 정함. 타겟팅은 매체 특성에 의존(예: 스포츠 잡지=남성)
```

이 구조의 핵심은 **거래 비용이 매우 높다**는 것이었다. 작은 광고주는 협상 테이블에 앉을 규모가 안 됐고, 매체사는 남는 지면(재고)을 헐값에라도 못 팔면 그냥 날렸다. 웹 광고의 역사는 결국 이 비효율을 자동화로 없애 온 과정이다.

---

## 2. 웹 배너의 탄생 (1994)

**1994년 10월, HotWired(와이어드 매거진의 웹사이트)에 걸린 AT&T의 468×60 배너**가 최초의 상업 웹 배너 광고로 꼽힌다. 문구는 단순했다 — "여기를 클릭해 본 적 있나요? 있을 겁니다."

이 배너의 클릭률은 무려 **44%**였다고 알려져 있다. 지금 기준(0.1% 안팎)으로는 상상하기 힘든 수치인데, 이유는 단순하다 — 사람들이 "클릭"이라는 행위 자체를 처음 경험했기 때문이다. 신기함이 곧 성과였던 시절이다.

초기 배너 거래도 결국 전통 광고와 같은 방식이었다 — **영업 담당이 사이트 트래픽을 근거로 광고주를 설득해 직접 판매**했다. 다만 지면이 물리적으로 유한하지 않다는 게 달랐다. 페이지뷰가 늘어날수록 팔 수 있는 "지면"도 무한히 늘어났다.

---

## 3. 검색 광고 — 구글이 규칙을 다시 쓰다

배너가 "브랜드를 보여주는 광고"였다면, 검색 광고는 **"의도를 가진 사람에게 정확히 필요한 순간 보여주는 광고"**였다. 이 전환이 산업의 방향을 완전히 틀었다.

### 최초는 구글이 아니다

유료 검색의 원조는 1998년 **GoTo.com**(이후 Overture, 야후에 인수)이다. 검색어별로 광고주가 입찰해 순위를 사는 방식을 처음 상용화했다.

### 구글 애드워즈 (2000) — 경매를 자동화하다

구글은 여기에 결정적인 개선을 더했다 — **입찰가 × 광고 품질(클릭률 예측)**로 순위를 매기는 방식이다. 돈만 많이 낸다고 1등이 아니라, 사람들이 실제로 클릭할 만한 광고여야 순위가 올라갔다. 이 설계 덕분에 검색 결과의 품질을 해치지 않으면서도 수익을 극대화할 수 있었다.

```
검색 광고 순위 결정 (구글 애드워즈 방식):
  광고 순위 = 입찰가 × 품질 점수(예상 클릭률·광고 관련성·랜딩페이지 경험)
  → 돈이 전부가 아니라, "클릭될 만한 광고"가 이긴다
```

### 애드센스 (2003) — 검색 밖으로 확장

구글은 애드워즈의 광고주 풀을 자기 검색 결과 밖, 즉 **제3자 웹사이트(퍼블리셔)**에도 꽂아 넣는 애드센스를 내놨다. 이때부터 "구글에 검색하지 않아도 구글 광고를 본다"는 지금의 구조가 시작됐다.

## 4. 프로그래매틱의 시대 — 사람 대신 기계가 사고판다

2009년 전후로 등장한 **RTB(Real-Time Bidding, 실시간 입찰)**가 웹 광고 거래 방식을 다시 한번 뒤집었다. 이제 지면 하나, 사용자 한 명이 페이지를 여는 그 순간마다 경매가 열린다.

```
프로그래매틱 거래 흐름 (사용자가 페이지를 여는 순간):
  사용자 방문 → 퍼블리셔의 지면 정보가 SSP로 전달
    → SSP가 애드 익스체인지에 경매 요청
    → 여러 DSP가 광고주를 대신해 실시간 입찰
    → 가장 높은 입찰가(또는 낙찰 규칙)가 승리
    → 낙찰된 광고가 사용자 화면에 렌더링
  이 전체 과정이 페이지 로딩 중 약 100ms 안에 끝난다
```

| 용어 | 역할 |
|---|---|
| **SSP** (Supply-Side Platform) | 퍼블리셔(매체) 편에서 지면 재고를 경매에 내놓는 플랫폼 |
| **DSP** (Demand-Side Platform) | 광고주 편에서 예산과 타겟팅 조건에 맞춰 자동 입찰하는 플랫폼 |
| **애드 익스체인지** | SSP와 DSP를 연결하는 경매 시장 자체 |
| **DMP** (Data Management Platform) | 타겟팅에 쓸 사용자 데이터(관심사·행동 이력)를 모으고 정리 |

### 헤더 비딩 — 퍼블리셔의 반격

초기 프로그래매틱은 구글의 애드 익스체인지가 사실상 유일한 경매장이라 퍼블리셔가 가격 결정권을 거의 못 가졌다. **헤더 비딩(Header Bidding)**은 페이지 로딩 단계에서 여러 애드 익스체인지에 동시에 경매를 열어, 구글 하나에 의존하지 않고 지면 가격을 끌어올리는 기술이다. 퍼블리셔들이 구글의 독점적 위치에 저항한 대표적 사례다.

## 5. 현재의 표준 — 업계가 합의한 규칙들

### 규격과 프로토콜

- **IAB(Interactive Advertising Bureau)**: 배너 표준 규격(300×250, 728×90 등)을 정하는 업계 단체. 지금도 대부분의 배너 사이즈는 IAB 표준을 따른다
- **VAST/VPAID**: 동영상 광고의 재생·추적 방식을 정한 표준. 유튜브를 포함한 대부분의 동영상 광고가 이 규격을 따른다
- **OpenRTB**: RTB 경매에서 SSP와 DSP가 주고받는 데이터 형식을 정한 프로토콜. 서로 다른 회사의 플랫폼이 호환되게 만드는 뼈대다

### 쿠키의 종말과 프라이버시 샌드박스

지난 20년간 타겟팅의 핵심 기술은 **서드파티 쿠키**였다 — 사용자가 어느 사이트를 돌아다녔는지 추적해 관심사를 추정하는 방식이다. 하지만 애플 Safari(ITP)가 먼저 차단했고, 구글 크롬도 서드파티 쿠키를 단계적으로 폐지하는 중이다. 그 대체재로 구글이 제안한 것이 **프라이버시 샌드박스**로, 개인을 특정하지 않고 브라우저 안에서 관심사 그룹만 추정해 타겟팅하는 방식이다.

여기에 **GDPR(유럽)**, **CCPA(캘리포니아)** 같은 규제가 겹치면서, 산업 전체가 "개인 식별 추적" 중심에서 "동의 기반·집계 데이터 기반" 타겟팅으로 이동하는 큰 전환기에 있다.

## 6. 누가 시장을 쥐고 있는가

### 듀오폴리 — 구글과 메타

미국 디지털 광고 지출의 절반가량을 **구글과 메타 두 회사**가 가져간다. 두 회사의 결정적 강점은 같다 — **로그인된 사용자의 방대한 1차 데이터**다. 검색어(구글), 소셜 그래프와 관심사(메타) 모두 서드파티 쿠키 없이도 타겟팅이 가능한 자체 데이터다. 이걸 흔히 **"월드가든(Walled Garden)"**이라 부른다 — 데이터도, 경매도, 지면도 전부 자기 울타리 안에서 끝나 외부 애드테크가 끼어들 틈이 없다.

### 구글의 수직계열화 — 그리고 반독점 소송

구글은 프로그래매틱 스택의 거의 모든 층을 소유한다.

```
구글의 애드테크 스택 장악:
  퍼블리셔 쪽 SSP  → Google Ad Manager
  경매 시장(익스체인지) → AdX (Google Ad Exchange)
  광고주 쪽 DSP    → Google Ads / DV360
  → 사실상 "경매 주최자이자 양쪽 대리인"인 구조
```

미국 법무부(DOJ)는 이 구조 자체가 독점이라며 2023년 소송을 제기했고, 2024년 연방 법원은 구글이 퍼블리셔용 애드테크 시장에서 불법적으로 독점을 유지했다고 판결했다. 애드테크 산업 구조를 뒤흔들 수 있는 사안으로 계속 진행 중이다.

### 떠오르는 3강 — 아마존과 리테일 미디어

아마존은 검색·소셜만큼 강력한 무기를 갖고 있다 — **실제 구매 데이터**다. "이 사람이 무엇을 봤는가"가 아니라 "이 사람이 실제로 무엇을 샀는가"를 안다는 건 타겟팅과 효과 측정 양쪽에서 압도적으로 유리하다. 이 흐름을 **리테일 미디어(Retail Media)**라 부르며, 월마트·쿠팡 같은 유통사들도 같은 모델을 빠르게 따라가는 중이다.

## 7. 돈은 어떻게 흐르는가 — 수익 구조

### 과금 모델

| 모델 | 과금 기준 | 주 용도 |
|---|---|---|
| **CPM** (Cost Per Mille) | 노출 1,000회당 과금 | 브랜드 인지도 광고 |
| **CPC** (Cost Per Click) | 클릭 1회당 과금 | 검색 광고, 트래픽 유도 |
| **CPA** (Cost Per Action) | 회원가입·구매 등 실제 행동당 과금 | 성과 기반 광고 |
| **CPI** (Cost Per Install) | 앱 설치 1건당 과금 | 모바일 앱 마케팅 |

광고주 입장에서는 "결과에 가까운 지표일수록" 신뢰하기 쉽지만, 매체사 입장에서는 "결과"는 매체가 통제할 수 없는 변수라 CPM을 선호한다. 이 줄다리기가 산업 전체의 가격 협상 구조를 만든다.

### 애드테크의 "중간 마진"

프로그래매틱 거래 1달러 중 퍼블리셔(콘텐츠를 만든 매체)에게 실제로 도달하는 비율은 업계 조사마다 다르지만 대체로 **50~70% 수준**으로 추정된다. 나머지는 SSP·애드 익스체인지·DSP·DMP 등 중간 사업자들이 각자 수수료로 가져간다. 이걸 흔히 **"애드테크 세금(Ad Tech Tax)"**이라 부르며, 퍼블리셔들이 헤더 비딩이나 자체 1차 데이터 강화로 이 마진을 줄이려는 이유이기도 하다.

```
프로그래매틱 1달러의 흐름 (개념도):
  광고주 지출 100
    → DSP 수수료
    → 애드 익스체인지 수수료
    → SSP 수수료
    → 퍼블리셔 실수령 ~50~70
  월드가든(구글 검색·메타 피드)은 이 체인이 훨씬 짧다 → 마진율이 훨씬 높음
```

이 마진 구조의 차이가 왜 구글·메타가 "당연히" 더 유리한 사업 모델인지를 설명한다 — 외부 중개자에게 나눠줄 몫이 애초에 거의 없다.

## 정리

```
기원          → 협상 기반 지면 판매 (신문·TV·라디오)
1994          → 최초의 웹 배너(AT&T, HotWired) — 영업 담당이 직접 판매
2000          → 구글 애드워즈 — 입찰가×품질점수 경매로 검색 광고 자동화
2003          → 애드센스 — 검색 밖 제3자 사이트로 광고 확장
2009~         → RTB 프로그래매틱 — SSP·DSP·애드익스체인지가 100ms 안에 실시간 경매
현재 표준      → IAB 배너규격, VAST 동영상표준, OpenRTB 프로토콜, 서드파티 쿠키 폐지+프라이버시 샌드박스
누가 쥐고 있나  → 구글·메타 듀오폴리(월드가든), 구글의 스택 수직계열화(반독점 소송 중), 아마존 리테일미디어 부상
수익구조       → CPM/CPC/CPA/CPI 과금, 프로그래매틱은 중간 마진(애드테크 세금)으로 퍼블리셔 실수령 50~70%
```

웹 광고 역사를 관통하는 한 줄은 결국 "거래 비용을 얼마나 줄이느냐"다. 사람이 협상하던 것을 경매가 대체했고, 익명의 페이지뷰를 팔던 것을 식별된 사용자 타겟팅이 대체했다. 지금 벌어지는 서드파티 쿠키 폐지와 반독점 소송은 이 흐름의 다음 국면이다 — 프라이버시 규제가 타겟팅 정밀도를 다시 낮추는 동시에, 자체 데이터를 가진 소수(구글·메타·아마존)의 힘은 오히려 더 세지는 역설적인 방향으로 가고 있다.
