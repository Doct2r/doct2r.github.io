"AI 탑재 제품", "ML 모델 적용", "딥러닝 기반", "LLM 연동" — 뉴스와 제품 소개에서 이 단어들이 뒤섞여 쓰인다. 같은 말인지, 다른 말인지, 어느 쪽이 더 큰 개념인지 불분명하다. 이것을 명확히 하려면 AI가 어떻게 시작됐고 어떻게 분기했는지를 역사 순으로 따라가야 한다.

---

## 1. 분류 체계부터 — 포함 관계

결론을 먼저 제시하고 이유를 설명한다.

```
AI (인공지능)
└── 기호주의 AI (규칙 기반, 전문가 시스템)
└── 머신러닝 (ML)
    └── 딥러닝 (Deep Learning)
        └── LLM (Large Language Model)
        └── CNN, RNN, GAN, Diffusion ...
    └── 결정 트리, SVM, 랜덤 포레스트 ...
└── 탐색/최적화 (A*, 유전 알고리즘 ...)
└── 강화학습 (RL) — ML과 겹치기도 함
```

LLM ⊂ 딥러닝 ⊂ ML ⊂ AI. ML을 "AI"라고 부르는 것은 틀리지 않다. ML은 AI의 하위 집합이니까. 다만 역은 성립하지 않는다. AI가 전부 ML은 아니다.

"같은 계통인가?" 라는 질문에 답하면 — **같은 이름(AI)을 쓰지만, 접근 철학이 근본적으로 다른 두 계통**이 AI 안에 공존하고 있다. 역사가 그것을 보여준다.

## 2. 탄생 — 1950년대

### 앨런 튜링과 질문 하나

1950년, 앨런 튜링이 논문 "Computing Machinery and Intelligence"에서 이 질문을 던졌다.

> "기계가 생각할 수 있는가?"

그가 제안한 **튜링 테스트**: 인간 심판이 채팅으로 상대를 보지 않고 대화할 때, 기계를 인간으로 오인하면 그 기계는 지능을 가진 것으로 본다.

이것이 AI 분야의 시작점이다. 아직 컴퓨터가 진공관으로 돌아가던 시절이었다.

### 다트머스 회의 (1956) — "인공지능"이라는 이름의 탄생

1956년 여름, 존 매카시(John McCarthy)가 다트머스 대학에서 워크숍을 열었다. 이 자리에서 처음으로 **"Artificial Intelligence"(인공지능)**라는 용어가 공식화됐다.

참석자들: 존 매카시, 마빈 민스키, 클로드 섀넌(정보 이론의 아버지), 허버트 사이먼

선언: "학습의 모든 측면, 또는 지능의 모든 특성은 원칙적으로 기계가 시뮬레이션할 수 있도록 정밀하게 기술될 수 있다."

이때의 낙관론은 상당했다. "20년 안에 인간 수준의 기계 지능이 가능하다." 실제로는 20년이 아니라 70년 뒤에도 논쟁 중이다.

## 3. 첫 번째 계통 — 기호주의 AI (1950s~1980s)

### 접근 철학: "지능은 논리와 규칙이다"

초기 AI 연구자들은 인간의 지능이 **기호 조작(Symbol Manipulation)**이라고 믿었다. 언어는 기호고, 추론은 기호의 논리적 연산이다. 따라서 기계도 충분히 많은 규칙과 논리만 갖추면 지능적으로 행동할 수 있다.

```
기호주의 AI의 구조:
  지식 베이스 (Knowledge Base)
  + 추론 엔진 (Inference Engine)
  → 결론 도출
```

### 초기 성과들

**Logic Theorist (1956)**: 허버트 사이먼과 앨런 뉴웰이 만든 최초의 AI 프로그램. 수학 정리를 자동으로 증명했다. "Principia Mathematica"의 52개 정리 중 38개를 증명.

**General Problem Solver (1957)**: 범용 문제 해결기. 목표 상태와 현재 상태의 차이를 줄이는 방식으로 문제를 풀었다. 체스, 퍼즐, 수학 증명에 적용.

**LISP (1958)**: 존 매카시가 AI 연구를 위해 설계한 프로그래밍 언어. 기호 처리에 특화. 현재도 Clojure, Racket 형태로 살아있다.

### 전문가 시스템 (Expert Systems, 1970s~1980s)

1970년대에 기호주의 AI의 상업적 구현이 등장했다.

**MYCIN (1974)**: 스탠퍼드에서 개발한 의료 진단 시스템. 600개 이상의 IF-THEN 규칙으로 혈액 감염 질환을 진단했다. 전문 의사와 비슷한 수준의 진단 정확도를 보여 충격을 줬다.

```
MYCIN 규칙 예시:
IF   환자의 감염 유형이 박테리아성이고
AND  환자가 면역 억제 상태이고
AND  감염 부위가 혈액이면
THEN Pseudomonas aeruginosa 가능성 0.6
```

**DENDRAL**: 화합물의 분자 구조를 분석하는 시스템. 화학자 수준의 추론.

**R1/XCON (1980)**: DEC(Digital Equipment Corporation)의 컴퓨터 주문 설정 시스템. 연간 4천만 달러의 비용을 절감.

이 시기 전문가 시스템은 실제 산업에서 쓰인 최초의 상업적 AI였다.

## 4. 첫 번째 AI 겨울 (1974~1980)

### 한계가 드러나다

기호주의 AI에는 근본적 문제가 있었다.

**프레임 문제(Frame Problem)**: 세상이 조금 바뀌면 관련된 모든 규칙을 업데이트해야 한다. 현실 세계의 변화를 규칙으로 따라잡는 게 불가능에 가깝다.

**조합 폭발(Combinatorial Explosion)**: 체스 같은 게임에서 경우의 수가 기하급수적으로 늘어나 탐색이 불가능해진다.

**상식 추론 불가**: 인간이 당연하게 아는 것("컵을 떨어뜨리면 깨진다")을 기계는 규칙으로 명시해야 한다. 그 규칙의 수가 수백만 개에 달했고 끝이 없었다.

1973년 영국에서 **라이트힐 보고서(Lighthill Report)**가 발표됐다. "AI는 과장됐다. 실용적 성과가 없다." 이후 영국 정부의 AI 연구 지원이 대폭 삭감됐다. 미국도 비슷한 흐름으로 첫 번째 **AI 겨울**이 찾아왔다.

## 5. 두 번째 AI 겨울 (1987~1993)

전문가 시스템 붐이 꺼지면서 두 번째 겨울이 왔다.

**유지보수 악몽**: MYCIN이 600개 규칙으로 동작한다면, 실제 대형 시스템은 수만 개 규칙이 필요했다. 규칙 간 충돌, 예외 처리, 업데이트 비용이 감당 불가능했다.

**지식 획득 병목(Knowledge Acquisition Bottleneck)**: 전문가의 암묵적 지식을 규칙으로 변환하는 작업이 극도로 어렵고 비쌌다.

**LISP 기계 시장 붕괴**: AI 전용 하드웨어인 LISP machine 시장이 일반 워크스테이션에 밀려 무너졌다.

이 시기 AI 연구비가 급감했다. 많은 AI 회사가 폐업했다.

## 6. 두 번째 계통 — 머신러닝의 부상 (1980s~2000s)

### 철학적 전환: "규칙을 짜지 말고 데이터에서 배우게 하자"

기호주의의 실패에서 다른 계통이 자랐다. **연결주의(Connectionism)** 또는 **통계적 접근**이었다.

핵심 질문의 전환:
- 기호주의: "어떻게 지식을 규칙으로 표현하는가?"
- 머신러닝: "어떻게 데이터에서 패턴을 스스로 찾게 하는가?"

### 신경망의 기원 — McCulloch-Pitts 뉴런 (1943)

사실 신경망의 개념은 1943년으로 거슬러 올라간다. 워런 매컬러치와 월터 피츠가 뇌의 뉴런을 수학 모델로 표현했다.

```
뉴런 모델:
  입력 신호들의 가중합 → 활성화 함수 → 출력
  if (가중합 > 임계값) → 발화(1)
  else → 비발화(0)
```

**퍼셉트론(Perceptron, 1957)**: 프랭크 로젠블라트가 단층 신경망을 구현. 이미지에서 직선을 학습해 분류. "기계가 스스로 학습한다"는 개념의 최초 시연.

그러나 1969년 마빈 민스키와 시모어 페퍼트가 **"퍼셉트론"** 책에서 XOR 문제를 단층 신경망이 풀 수 없음을 수학적으로 증명. 신경망 연구가 한동안 침체됐다.

### 역전파(Backpropagation)의 부활 (1986)

1986년 제프리 힌턴(Geoffrey Hinton), 데이비드 루멜하트, 로널드 윌리엄스가 **역전파 알고리즘**을 발표했다.

```
역전파의 원리:
  1. 입력 → 신경망 → 예측 출력 (순전파)
  2. 예측과 정답의 차이(오류) 계산
  3. 오류를 네트워크 역방향으로 전달
  4. 각 가중치를 오류 감소 방향으로 조금씩 수정
  5. 반복 → 점점 정확해짐
```

다층 신경망을 학습시킬 수 있게 됐다. XOR 문제도 해결됐다.

### 머신러닝의 다양한 알고리즘 (1990s~2000s)

신경망 외에도 다양한 ML 접근이 발전했다.

**SVM (Support Vector Machine)**: 블라디미르 바프닉이 1990년대 개발. 고차원 공간에서 데이터를 가르는 최적 경계선을 찾는다. 이론이 탄탄하고 소량 데이터에서도 성능이 좋아 한동안 신경망보다 더 주목받았다.

**결정 트리 / 랜덤 포레스트**: 데이터를 IF-THEN으로 분기해 나가는 트리 구조. 해석 가능성이 높다. 랜덤 포레스트는 수천 개의 트리를 앙상블.

**나이브 베이즈**: 스팸 필터링의 핵심 알고리즘. 이메일 단어들의 확률로 스팸 여부를 판별.

이 알고리즘들은 지금도 쓰인다. 데이터가 적거나, 해석이 필요하거나, 간단한 분류 작업에서 딥러닝보다 실용적이다.

## 7. 딥러닝 혁명 (2006~2012)

### 왜 딥러닝이 부활했나

다층 신경망(깊은 신경망 = Deep Neural Network)은 이론적으로 강력했지만 두 가지 문제가 있었다.

1. **기울기 소실(Vanishing Gradient)**: 역전파할 때 오류 신호가 층을 거치며 사라져 깊은 망을 학습시킬 수 없었다
2. **데이터 부족**: 수십만 개 이상의 데이터가 필요한데 없었다
3. **연산 부족**: 엄청난 행렬 연산이 필요한데 CPU로는 불가능했다

2000년대 후반, 세 가지 조건이 동시에 해결됐다.

- **데이터**: 인터넷 보급 → 이미지, 텍스트가 폭발적으로 생성. ImageNet 데이터셋(140만 장 이미지)
- **연산**: GPU의 병렬 처리가 신경망 행렬 연산에 완벽히 맞음. CUDA 등장(2007)
- **알고리즘**: ReLU 활성화 함수, Dropout 등으로 기울기 소실 문제 완화

### AlexNet — 게임 체인저 (2012)

2012년 ImageNet 이미지 분류 대회(ILSVRC). 제프리 힌턴 팀의 **AlexNet**이 오류율 15.3%로 우승했다. 2위는 26.2%였다. 10%포인트 이상의 압도적 격차.

AlexNet은 CNN(합성곱 신경망)을 GPU로 학습시킨 딥러닝 모델이었다. 이 결과 이후, 이미지 인식 분야가 딥러닝으로 완전히 전환됐다.

```
딥러닝의 구조적 특징:
  입력층 → [은닉층] → [은닉층] → ... → 출력층
           (특징 추출)  (추상화)

  예: 이미지 → 엣지 감지 → 형태 인식 → 물체 분류
```

층이 깊어질수록(Deep) 더 추상적인 특징을 학습한다. 이것이 기존 ML과 구별되는 딥러닝의 핵심이다. 특징(Feature)을 사람이 정의하지 않아도 된다.

## 8. LLM의 시대 — Transformer (2017~현재)

### RNN의 한계

2010년대 중반, 텍스트와 시퀀스 데이터에는 **RNN(순환 신경망)**과 **LSTM**이 쓰였다. 이전 단어를 기억하며 다음 단어를 예측하는 구조였다.

문제가 있었다.

- **순차 처리**: 단어를 하나씩 처리해야 해서 병렬화 불가. 학습이 느렸다
- **장거리 의존성**: 문장이 길어지면 앞의 내용을 기억하지 못했다

### "Attention is All You Need" (2017)

2017년 구글 브레인이 발표한 논문 제목이다. **Transformer** 아키텍처를 제안했다.

핵심 아이디어: **어텐션(Attention)** 메커니즘. 문장의 모든 단어가 다른 모든 단어와 동시에 관계를 계산한다.

```
RNN 방식 ("나는 밥을 먹었다"):
  나는 → 밥을 → 먹었다 (순서대로 처리)

Transformer 방식:
  나는 ←→ 밥을 ←→ 먹었다 (동시에 모든 관계 계산)
  "먹었다"는 "나는"과 "밥을" 모두와 관계를 동시에 파악
```

순차 처리 없이 병렬 처리가 가능해져 GPU를 100% 활용할 수 있게 됐다. 더 크고 깊은 모델을 학습할 수 있었다.

### GPT 계보 — 스케일의 법칙

OpenAI는 Transformer를 기반으로 **GPT(Generative Pre-trained Transformer)** 시리즈를 개발했다.

| 모델 | 연도 | 파라미터 수 | 비고 |
|---|---|---|---|
| GPT-1 | 2018 | 1.17억 | 텍스트 생성 가능성 증명 |
| GPT-2 | 2019 | 15억 | "너무 위험하다"며 처음엔 비공개 |
| GPT-3 | 2020 | 1,750억 | 프롬프트만으로 다양한 작업 가능 |
| GPT-4 | 2023 | 미공개 (추정 1조+) | 멀티모달, 추론 능력 대폭 향상 |
| ChatGPT | 2022 | GPT-3.5 기반 | RLHF로 대화 튜닝 → 대중화 |

**스케일링 법칙(Scaling Law)**: 모델 크기, 데이터 양, 컴퓨팅을 늘리면 성능이 예측 가능하게 향상된다는 경험 법칙. "크면 클수록 똑똑해진다"는 것이 경험적으로 증명됐다.

### LLM이란

**LLM(Large Language Model)** = 방대한 텍스트로 사전 학습된 대형 언어 모델.

- **Pre-training**: 인터넷의 수조 개 토큰(단어 조각)으로 "다음 단어 예측" 학습
- **Fine-tuning**: 특정 목적(대화, 코드, 요약)에 맞게 추가 학습
- **RLHF**: 인간 피드백 기반 강화학습으로 응답 품질 조정

현재 주요 LLM:
- OpenAI: GPT-4o, o1, o3
- Anthropic: Claude 3.5/4 시리즈
- Google: Gemini 1.5/2.0
- Meta: Llama 3.x (오픈소스)

## 9. 실제로 같은 건가 — 두 계통의 비교

AI 역사에서 두 가지 근본적으로 다른 접근이 충돌하고 공존해왔다.

| | 기호주의 AI | 연결주의 AI (ML/DL) |
|---|---|---|
| 핵심 개념 | 기호, 규칙, 논리 | 통계, 패턴, 가중치 |
| 지식 획득 | 전문가가 규칙 작성 | 데이터에서 스스로 학습 |
| 해석 가능성 | 높음 (규칙이 명시적) | 낮음 (블랙박스) |
| 소량 데이터 | 강함 | 약함 |
| 대량 데이터 | 약함 | 강함 |
| 새 도메인 적응 | 규칙 재작성 필요 | 데이터만 바꾸면 됨 |
| 대표 실패 | 전문가 시스템 유지비 | 환각(Hallucination) |

현재 주류는 연결주의(딥러닝)다. 그러나 기호주의가 완전히 사라진 건 아니다.

**뉴로-기호주의(Neuro-Symbolic AI)**: 두 접근의 장점을 결합하려는 연구. LLM의 언어 능력 + 논리 추론 엔진을 결합해 환각을 줄이는 방향.

## 10. AI의 범위 문제 — 뭐가 AI이고 뭐가 아닌가

### 좁은 AI vs 범용 AI

| 구분 | 설명 | 현황 |
|---|---|---|
| **ANI** (Narrow AI) | 특정 작업만 잘하는 AI | 현재 모든 AI |
| **AGI** (General AI) | 인간처럼 어떤 작업이든 | 미달성 |
| **ASI** (Super AI) | 인간을 초월하는 AI | 이론상 개념 |

현재의 GPT-4, Claude, AlphaGo — 전부 ANI다. GPT-4가 대화를 잘해도 자전거 타는 법을 모른다. AlphaGo는 바둑에서 세계 챔피언을 이겼지만 오목은 못 한다.

### "이것도 AI인가?" 체크리스트

흔히 혼동하는 사례들.

**체스/바둑 AI (Minimax, MCTS)**
→ AI가 맞다. 탐색 기반 AI. ML이나 딥러닝은 아닐 수 있다 (AlphaGo는 딥러닝 + 탐색).

**A* 길찾기 알고리즘**
→ 넓은 의미의 AI(지능적 문제 해결)이지만, 보통 AI라 부르지 않는다. 알고리즘이라 부른다.

**SPAM 필터 (나이브 베이즈)**
→ ML이므로 AI가 맞다. 하지만 딥러닝이나 LLM은 아니다.

**추천 시스템 (넷플릭스, 유튜브)**
→ ML (협업 필터링, 딥러닝). AI가 맞다.

**얼굴 인식 잠금**
→ 딥러닝 (CNN). AI가 맞다.

**ChatGPT**
→ LLM → 딥러닝 → ML → AI. 모두 맞다.

**if-else로 만든 챗봇**
→ 프로그래밍이다. AI가 아니다. 학습도 없고 패턴도 없다.

### ML과 AI를 혼용하는 이유

마케팅에서 "AI"는 "ML"을 대신하는 단어로 쓰인다. "ML 탑재 카메라"보다 "AI 카메라"가 소비자에게 더 직관적이기 때문이다. 틀린 말은 아니지만 정확한 말도 아니다.

개발자와 엔지니어 사이에서는 어느 수준의 기술을 쓰느냐가 중요하다.

```
실무에서의 정확한 표현:
  "SVM 기반 분류기"  → ML, 딥러닝 아님
  "CNN 기반 이미지 분류" → 딥러닝
  "Transformer 기반 텍스트 생성" → 딥러닝, LLM 계열
  "GPT-4 API 연동" → 외부 LLM 활용
```

## 정리 — 한 장의 지도

```
연도      사건                      계통
─────────────────────────────────────────────────
1950  튜링 테스트 제안              AI 개념의 탄생
1956  다트머스 회의, "AI" 명명      기호주의 AI 시작
1957  퍼셉트론                     연결주의 (신경망) 씨앗
1965  전문가 시스템 연구 시작       기호주의 全盛期
1969  XOR 문제 → 신경망 침체       연결주의 겨울
1974  첫 번째 AI 겨울              기호주의 퇴조
1980  전문가 시스템 상업화          기호주의 부활
1986  역전파 알고리즘              연결주의 부활
1987  두 번째 AI 겨울              기호주의 퇴조(최종)
1990s SVM, 랜덤 포레스트           통계적 ML 부상
2006  딥러닝 사전학습 기법          딥러닝 부활
2012  AlexNet, ImageNet 압도       딥러닝 주류화
2017  Transformer 논문             LLM 시대의 씨앗
2020  GPT-3                       스케일의 법칙 증명
2022  ChatGPT                     AI 대중화
2024~ LLM 멀티모달, 에이전트       현재진행형
```

AI는 단일한 기술이 아니다. 70년 동안 두 철학이 경쟁하고 합쳐진 역사다. 지금 우리가 "AI"라고 부르는 것의 대부분은 연결주의 계통의 딥러닝, 그 중에서도 Transformer 기반 모델이다. 그러나 기호주의의 접근 — 명시적 규칙, 논리 추론 — 은 LLM의 한계(환각, 수학 오류, 논리 오류)를 보완하는 방향으로 다시 연구되고 있다. 두 계통은 같은 목표를 향해 다른 길을 걸어온 셈이다.
