[HTML이 SGML에서 영감받아 탄생한 과정](/posts/markup-language-history/)은 이미 다룬 적이 있다. 그런데 그 이후 HTML의 역사에는 훨씬 극적인 사건이 하나 숨어 있다 — 지금 우리가 쓰는 HTML5는 사실 공식 표준화 기구인 W3C가 만든 게 아니다. W3C는 한때 HTML을 완전히 갈아엎으려다 실패했고, 정작 지금의 HTML5를 실제로 만든 건 그 실패에 반발해 뛰쳐나온 반란 그룹이었다.

---

## 1. HTML+와 HTML 2.0 — 첫 진짜 표준화

데이브 래겟은 1993년 표와 개선된 폼 기능 등을 담은 "HTML+"라는 비공식 제안을 냈다 — 이 자체가 공식 표준이 되진 않았지만 이후 표준화 작업에 흡수됐다. HTML 2.0은 1995년 11월 IETF RFC 1866으로 나왔는데, 이건 당시 브라우저들이 이미 쓰고 있던 관행(폼 기능 포함)을 성문화한, 진짜 표준으로 취급된 첫 HTML 스펙이었다.

## 2. HTML 3.2와 HTML 4 — 브라우저 전쟁을 뒤늦게 추인하다

HTML 3.2는 1997년 1월 14일 W3C 권고안이 됐다. 이건 수학 마크업 같은 야심 찬 기능을 담았던 이전의 "HTML 3.0" 초안이 무산된 뒤 나온 축소판이었는데, 실제로는 넷스케이프와 인터넷 익스플로러가 브라우저 전쟁 시기에 이미 구현해둔 기능(표·애플릿·이미지 둘레로 흐르는 텍스트 등)을 새로 개척한 게 아니라 뒤늦게 표준으로 추인한 것에 가까웠다. HTML 4.0은 1997년 12월 18일, HTML 4.01은 1999년 12월 W3C 권고안이 됐다 — Strict·Transitional·Frameset 세 가지 DTD 변형을 두고, CSS로 표현을 분리하려는 움직임이 본격화됐다.

## 3. XHTML 1.0과 실패한 XHTML 2.0

XHTML 1.0은 2000년 1월 26일 W3C 권고안이 됐다 — HTML 4.01을 엄격하게 정형(well-formed)된 XML 응용 프로그램(닫는 태그·소문자 요소·따옴표 속성 필수)으로 재구성한 것이었다. 그런데 W3C는 여기서 멈추지 않았다. XHTML 2.0은 2002년 8월부터 2006년 7월 사이 여덟 개의 작업 초안이 나왔는데, 명시적으로 "완전한 단절"을 목표로 하위 호환성을 의도적으로 포기했다. 구체적으로 알려진 비호환 설계로는, `<img>` 요소를 범용 `<object>` 스타일로 대체하려 한 것(훗날 초안에서 비호환 `<img>`가 다시 등장했다)과, `href` 같은 링크 속성을 `<a>` 태그에만 국한하지 않고 거의 모든 요소에 허용하려 했던 일반화 시도가 있다 — 이게 웹 개발자와 브라우저 벤더 사이에서 실제로 경보를 울린 논란거리였다. 결국 2009년 W3C는 XHTML 2.0 작업그룹의 헌장이 만료되도록 두며 HTML5를 차세대 유일 표준으로 사실상 인정했다 — XHTML 2.0은 정식 권고안 단계에는 끝내 이르지 못했다.

## 4. WHATWG 반란 — 진짜 핵심

2004년 6월 W3C의 "웹 애플리케이션과 복합 문서 워크숍"에서 오페라와 모질라가 공동으로 낸 제안서 — 기존 HTML을 점진적이고 하위 호환되게 확장하자는, 훗날 "웹 폼 2.0"의 씨앗이 된 안건 — 가 참석자 51명 앞에서 표결에 부쳐졌다. 8명이 찬성, 11명이 반대했고, 재구성된 동의안조차 14명이 반대해 부결됐다. W3C 참석자 대부분은 그 대신 XML 기반의 XHTML 2.0 방향을 계속 밀고 싶어 했다.

그런데 불과 이틀 뒤인 2004년 6월 4일, WHATWG(Web Hypertext Application Technology Working Group) 메일링 리스트가 발표됐다. 애플·모질라 재단·오페라 소프트웨어 소속 엔지니어들이 회사의 공식 입장이 아니라 개인 자격으로 모여 결성한 연합이었고, 당시 오페라에 있던 이언 히크슨이 이 그룹을 이끌었다. 첫 구체적인 목표는 표결에서 부결됐던 바로 그 "웹 폼 2.0"이었다.

## 5. 2006년의 뒤집기

2006년 10월 W3C 디렉터 팀 버너스리는 "HTML 재발명"이라는 블로그 글을 발표했다 — XML·XHTML 2.0으로 옮겨가는 경로가 "그냥 작동하지 않았다"고 인정하며 "HTML을 점진적으로 진화시킬 필요가 있다"고 선언한 것이다. HTML을 XHTML 2.0으로 완전히 대체하겠다던 W3C의 기존 입장에서 완전히 돌아선 순간이었다. 몇 달 뒤 W3C는 새로운 HTML 작업그룹 헌장을 발행하며, 처음부터 다시 시작하는 대신 WHATWG가 이미 만들어놓은 산출물을 토대로 삼기로 했다.

## 6. HTML5 표준화 이정표

HTML5의 첫 공개 작업 초안은 2008년 1월 22일 W3C에서 나왔다 — 편집자는 구글의 이언 히크슨과 애플의 데이비드 하이엇이었다. 그런데 HTML5가 정식으로 W3C 권고안 지위를 얻은 건 2014년 10월 28일이었다 — 이미 수년간 실제 브라우저에서 광범위하게 쓰이고 있던 걸 감안하면 놀랍도록 늦은 "공식" 표준화 시점이었다.

## 7. "리빙 스탠더드" 합의

2019년 5월 28일 W3C와 WHATWG는 양해각서를 체결했다 — WHATWG가 계속 갱신하는 "HTML 리빙 스탠더드"가 유일한 규범적 출처가 되고, W3C는 자체적으로 버전을 나눈 HTML5.x 권고안을 더는 따로 발행하지 않고 WHATWG의 리뷰 초안을 그대로 W3C 권고안으로 끌어올리는 역할만 맡기로 했다. 명시적인 이유는 서로 경쟁하는 두 개의 "규범적" 스펙이 존재하는 게 커뮤니티에 해롭다는 것이었다.

---

## 정리

"HTML5는 W3C가 만들었다"는 흔한 가정은 틀렸다. W3C는 2000년대 중반 XHTML 2.0이라는, 하위 호환성을 의도적으로 버린 급진적 재설계로 HTML을 완전히 대체하려 했다. 이 시도가 웹 개발자들의 반발과 현실적인 작동 불능으로 실패하자, 정작 실제 HTML5를 만든 건 W3C 바깥에서 애플·모질라·오페라 엔지니어들이 자발적으로 꾸린 WHATWG라는 반란 그룹이었다. W3C는 그 반란의 결과물을 뒤늦게 추인했을 뿐이고, 그 추인조차 2019년에야 하나의 표준으로 완전히 정리됐다.
