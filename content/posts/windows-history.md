Windows를 이해하려면 하나의 제품이 아니라 **두 개의 서로 다른 혈통이 병존하다 충돌하고 합쳐진 역사**로 봐야 한다. 하나는 DOS에서 자란 소비자 계열, 다른 하나는 처음부터 다시 설계된 NT 계열이다. 그 긴 역사 안에서 "퐁당퐁당"이라 불리는 성공과 실패의 교번이 생긴 것은 우연이 아니었다.

## 1. 서막 — DOS와 GUI 충격 (1981~1984)

### MS-DOS의 탄생과 한계

1981년 IBM이 PC를 출시하면서 Microsoft의 MS-DOS가 운영체제로 채택됐다. 원래는 Seattle Computer Products의 QDOS(Quick and Dirty OS)를 사와 손본 것이었다.

MS-DOS는 태생적 제약이 컸다.

```
MS-DOS의 구조적 한계:
  - 16비트 실주소 모드 (Real Mode)
  - 최대 1MB 메모리 주소 공간 (640KB가 실제 사용 가능)
  - 단일 태스킹 (한 번에 하나의 프로그램만)
  - 파일 시스템: FAT12/FAT16 (파일명 8.3 형식)
  - 보호 없음: 어떤 프로그램이든 메모리 전체에 접근
```

그런데 1984년 충격이 왔다. Apple이 **Macintosh**를 출시했다. GUI, 마우스, 아이콘, 창. Xerox PARC에서 온 개념들이 일반 소비자용 컴퓨터에 구현됐다. 빌 게이츠는 위기를 직감했다.

## 2. Windows 1.0 / 2.0 — GUI를 흉내 내다 (1985~1988)

### Windows 1.0 (1985) — 늦고, 어설프고, 필요했다

1981년 발표하겠다고 했으나 4년 지연된 끝에 나왔다. 초기 이름은 "Interface Manager"였다.

Windows 1.0은 운영체제가 아니었다. **MS-DOS 위에서 돌아가는 그래픽 셸(Shell)**이었다.

```
구조:
  하드웨어
  └── MS-DOS (실제 운영체제)
      └── Windows 1.0 (그래픽 인터페이스 레이어)
```

심각한 제약이 있었다. 창이 겹치지 않았다. 화면에서 타일처럼 나란히만 배치할 수 있었다. Apple이 특허를 이유로 겹치는 창을 소송으로 막았기 때문이라는 설이 있지만, 실제로는 합의 조건의 결과였다. 시장에서 외면받았다.

**탄생 이유**: Apple Mac의 GUI에 대한 대응. PC 시장을 잃지 않으려는 생존 전략.
**끝난 이유**: 태생적 한계. 다음 버전이 출시됐다.

### Windows 2.0 (1987) — 겹치는 창의 등장

창 겹침이 허용됐다. 키보드 단축키, 확장 메모리(EMS) 지원. Excel과 Word가 Windows용으로 나왔다. 여전히 틈새 제품이었지만 방향이 잡혔다.

## 3. Windows 3.x — 첫 번째 상업적 성공 (1990~1994)

### Windows 3.0 (1990) — 시장이 열리다

**보호 모드(Protected Mode)** 지원이 핵심이었다. Intel 286/386 프로세서의 보호 모드를 활용해 1MB 이상의 메모리를 사용할 수 있게 됐다. 프로그램 아이콘, 파일 관리자, 프린트 관리자. GUI가 쓸 만해졌다.

출시 2개월 만에 100만 카피가 팔렸다. PC 역사상 최초의 소프트웨어 블록버스터였다.

### Windows 3.1 (1992) — 진짜 시장 표준

- **TrueType 폰트**: Apple과 라이선스 계약. 화면과 프린트가 동일하게 보인다
- **멀티미디어**: 사운드카드, CD-ROM 지원
- **OLE (Object Linking and Embedding)**: 프로그램 간 데이터 공유
- **협력적 멀티태스킹(Cooperative Multitasking)**: 여러 프로그램을 동시에 열 수 있었다

그러나 "협력적" 멀티태스킹의 의미는, 각 프로그램이 스스로 CPU를 양보해야 한다는 뜻이었다. 프로그램 하나가 버그로 멈추면 전체가 멈췄다.

```
협력적 멀티태스킹의 문제:
  프로그램 A가 실행 중
  → 스스로 "yield" 신호를 보내야 B가 실행됨
  → A가 무한 루프에 빠지면? 전체 시스템이 응답 없음
```

**존재의 가치**: PC 시장의 주류가 됐다. DOS의 명령어 세계에서 GUI 세계로의 전환점.
**왜 끝났나**: 협력적 멀티태스킹의 불안정성, 16비트 한계. NT 계열과 Win95가 대체.

## 4. NT 커널의 탄생 — 또 다른 혈통의 시작 (1988~1993)

3.x가 DOS 위에서 그래픽을 그리는 동안, Microsoft는 내부에서 완전히 다른 운영체제를 설계하고 있었다.

### OS/2의 파국과 Dave Cutler

1987년 Microsoft와 IBM은 DOS의 후계자로 **OS/2**를 공동 개발하고 있었다. 그런데 Windows 3.0이 예상 외로 히트하면서 Microsoft의 전략이 흔들렸다.

1990년 Microsoft와 IBM의 파트너십이 결렬됐다. 각자의 길을 가기로.

빌 게이츠가 영입한 사람이 **Dave Cutler**였다. DEC(Digital Equipment Corporation)에서 VAX/VMS 운영체제를 설계한 전설적 시스템 프로그래머였다. 그는 OS/2와 DOS와 완전히 다른 새로운 커널을 만들었다.

### NT 커널의 설계 철학

```
Windows NT 커널 설계 원칙:
  이식성 (Portability)     : x86만이 아니라 MIPS, Alpha, PowerPC에서도 동작
  안정성 (Reliability)     : 프로그램 하나가 죽어도 시스템이 살아있음
  보안 (Security)          : C2 보안 등급 (미국 국방부 기준) 목표
  확장성 (Scalability)     : 서버부터 워크스테이션까지
```

**HAL (Hardware Abstraction Layer)**: 하드웨어 차이를 추상화하는 레이어. HAL만 바꾸면 다른 CPU 아키텍처에서 돌아갈 수 있다.

**선점형 멀티태스킹(Preemptive Multitasking)**: 운영체제가 CPU 스케줄링을 직접 통제한다. 프로그램이 협조하지 않아도 강제로 CPU를 빼앗을 수 있다.

**메모리 보호**: 각 프로세스가 독립된 가상 주소 공간을 가진다. 프로그램 A가 프로그램 B의 메모리를 건드릴 수 없다.

**링 구조 (Ring Architecture)**:
```
Ring 0 (커널 모드): 운영체제 코어, 드라이버
Ring 3 (사용자 모드): 일반 애플리케이션

Ring 3 프로그램이 하드웨어에 직접 접근하려 하면?
→ 시스템 콜을 통해 Ring 0에 요청
→ Ring 0이 검증 후 처리
→ 프로그램이 오작동해도 커널은 보호됨
```

**NTFS (New Technology File System)**: FAT의 후계자.
- 파일 크기 제한 없음 (FAT16은 2GB 제한)
- 저널링 (Journaling): 파일 변경 이력을 기록해 갑작스러운 전원 차단 후에도 파일 시스템 복구
- 파일/폴더 권한 (ACL)
- 파일명 최대 255자, Unicode 지원

### Windows NT 3.1 (1993) — NT의 첫 공개

버전 번호를 3.1로 맞춘 이유: 당시 Windows 3.1이 시장 표준이었기 때문에 사용자가 익숙하게 느끼도록.

그러나 NT 3.1은 **소비자용이 아닌 기업용**이었다. 워크스테이션과 서버 시장을 노렸다.

하드웨어 요구사항이 당시 기준으로 엄청났다 — 16MB RAM, 300MB 하드. 일반 PC에는 과분했다. 또한 많은 DOS용 프로그램이 하드웨어에 직접 접근했는데, NT의 Ring 구조 보호 때문에 동작하지 않는 경우가 많았다.

**탄생 이유**: 안정적이고 보안이 강한 기업용 OS. DOS의 한계를 완전히 벗어난 새 플랫폼.
**존재의 가치**: NT 커널의 모든 후속 Windows(XP, Vista, 7, 8, 10, 11)의 기반.

## 5. Win9x 혈통 — 소비자를 위한 진화 (1995~2000)

NT가 기업 시장을 공략하는 동안, DOS 기반 소비자 계열도 진화했다.

### Windows 95 (1995) — 시대를 바꾼 출시

**"Start Me Up"** — Rolling Stones의 곡을 광고에 쓰며 세계적 이벤트로 출시됐다. CNN이 실시간 중계했다. 빌 게이츠와 제이 레노가 투나잇쇼에 나왔다.

기술적으로 Win95는 절충의 산물이었다.

```
Windows 95의 내부:
  MS-DOS 7.0 (부팅 및 일부 레거시)
  + 32비트 Windows 코어 (새로 작성)
  = 혼합 아키텍처 (하이브리드)

  DOS 프로그램 → 여전히 동작
  Win16 프로그램 → 여전히 동작  
  Win32 프로그램 → 새로운 32비트 API
```

절충이었기에 NT만큼 안정적이지 않았다. 하지만 호환성이 핵심이었다.

**95가 가져온 것들:**
- **시작 버튼(Start Button)**: "Start"를 누르면 모든 것이 시작된다. 사용법을 이 버튼 하나가 가르쳤다
- **작업 표시줄(Taskbar)**: 실행 중인 프로그램 목록
- **탐색기(Explorer)**: 파일을 시각적으로 관리
- **Plug and Play**: 하드웨어를 꽂으면 드라이버가 자동 인식
- **긴 파일명**: 8.3 제한 해제. "내 문서.txt" 가능
- **DirectX**: 게임을 위한 하드웨어 직접 접근 API (이때부터 Windows가 게임 플랫폼이 됨)
- **TCP/IP 내장**: 인터넷 연결이 OS 기본 기능이 됨
- **Internet Explorer 1.0** (OSR2부터 IE 3.0): 인터넷의 대중화

**Registry(레지스트리)**: Win95부터 시스템 설정을 중앙 관리하는 계층적 데이터베이스로 전환. 이전에는 수십 개의 .INI 파일이 흩어져 있었다.

**탄생 이유**: NT는 기업용이었고 너무 무거웠다. 소비자 PC에서 DOS의 호환성을 유지하면서 GUI와 32비트 성능을 제공해야 했다.
**존재의 가치**: 인터넷 시대의 문을 연 대중적 OS. PC를 일반 가정에 보급한 원동력.
**약점**: DOS 레거시라는 원죄. 공유 메모리 공간 — 어느 드라이버 하나가 죽으면 전체가 죽는 BSOD.

### Windows 98 (1998) — 인터넷을 OS로 흡수하다

Win95의 업그레이드판이었다. 기술적으로 큰 변화보다 완성도를 높인 버전.

- **USB 지원**: 꽂으면 인식. 지금은 당연하지만 당시엔 혁신
- **FAT32**: FAT16의 2GB 파티션 제한 해제
- **IE4 통합**: Explorer 셸에 IE가 통합. 브라우저와 파일 탐색기가 하나가 됨. 이것이 반독점 소송의 핵심

**브라우저 전쟁**: Netscape Navigator를 죽이기 위해 IE를 무료로 번들했다. 1998년 미국 법무부가 반독점법 위반으로 제소. 2001년 합의로 마무리됐지만, Microsoft는 브라우저 시장을 장악했다.

**탄생 이유**: 인터넷 시대에 맞는 업그레이드. USB와 DVDs라는 새 하드웨어 지원.
**왜 끝났나**: Win9x 혈통의 한계 — DOS 레거시. 다음 ME에서 최악의 형태로 드러났다.

## 6. Windows ME — Win9x 혈통의 비극적 종언 (2000)

**ME(Millennium Edition)**는 Win9x 혈통의 마지막이자 가장 실패한 버전이다.

### 왜 만들었나

2000년, NT 기반의 Windows 2000이 출시됐다. 그런데 기업용이었고 게임 호환성 문제가 있었다. 소비자용 자리를 채워야 했다. 그것이 ME였다.

문제는 **이미 시대가 지난 아키텍처로 만들었다**는 것이다.

```
2000년의 상황:
  NT 커널 → 안정적, 기업용
  Win9x 커널 → DOS 레거시, 불안정, 이미 한계

Microsoft의 선택:
  소비자용 → Win9x를 기반으로 ME 출시 (단기 해결책)
  기업용   → 2000 출시

이상적이었다면:
  XP처럼 처음부터 NT 기반으로 소비자용을 만들었어야 했다
```

**ME가 가져온 기능들:**
- Windows Movie Maker: 비디오 편집
- Windows Media Player 7
- System Restore: 시스템을 이전 상태로 되돌리는 기능

**그러나:**
- System Restore가 오히려 문제를 만드는 경우가 많았다
- 실제 MS-DOS 모드를 제거 → DOS 게임이 안 되는 경우 발생
- BSOD(블루스크린)와 불안정성

ME는 시장에서 혹평을 받았다. "역대 최악의 Windows"라는 평가가 지금도 유효하다.

**탄생 이유**: Win9x 혈통으로 소비자 시장을 메꾸는 임시방편.
**존재의 가치**: 없다시피 했다. XP로 가기 전 공백을 채웠을 뿐.
**왜 끝났나**: NT 커널 기반 XP가 나오면서 Win9x 혈통 자체가 역사에서 사라졌다.

## 7. Windows 2000 — NT가 완성되다 (2000)

NT 4.0(1996)을 거쳐 Windows 2000이 나왔다. NT 커널의 완성형이었다.

- **Active Directory**: 기업 네트워크 사용자·컴퓨터 관리의 표준
- **Plug and Play**: NT 계열 최초로 본격 지원
- **NTFS 개선**: 암호화 파일 시스템(EFS), 디스크 쿼터
- **IIS 5.0**: 웹 서버
- **안정성**: ME와 비교할 수 없는 수준

그러나 문제가 있었다. 여전히 많은 소비자용 게임과 프로그램이 DOS에 직접 접근하려 해서 NT의 Ring 보호에 막혔다. DirectX 지원도 제한적이었다. 일반 소비자가 게임 PC로 쓰기엔 불편함이 있었다.

**탄생 이유**: NT 4.0의 완성형. 기업 인프라의 기반.
**존재의 가치**: Active Directory 도입으로 기업 네트워크 관리 표준을 정착시켰다.
**왜 끝났나**: XP가 소비자 + 기업을 모두 포괄하면서.

## 8. Windows XP — 두 혈통의 통합 (2001)

**XP는 Win9x와 NT 혈통이 하나로 합쳐지는 역사적 전환점이었다.**

"eXPerience(경험)"라는 이름처럼, XP는 사용자 경험에 집중했다.

### 통합의 의미

```
이전:
  소비자 → Win9x 계열 (95, 98, ME) — 호환성 좋지만 불안정
  기업   → NT 계열 (NT 4.0, 2000)  — 안정적이지만 호환성 부족

XP:
  NT 커널 위에 소비자 경험을 얹는다
  → 안정성 + 호환성
  → Win9x 계열의 DirectX, 게임 지원
  → NT의 메모리 보호, 선점형 멀티태스킹
```

**XP가 가져온 것들:**
- **ClearType**: LCD 모니터 최적화 폰트 렌더링
- **원격 데스크탑**: 내장
- **Windows XP SP2 (2004)**: 보안의 전환점. Windows Firewall 기본 활성화, DEP(Data Execution Prevention), 팝업 차단. 이것이 없었다면 인터넷 익스플로러 시대의 보안 재앙은 더 심각했을 것이다
- **드라이버 호환성**: Win9x 시대 드라이버도 많이 수용
- **Home / Professional 에디션**: 소비자와 기업 버전 분리

### XP의 긴 수명

XP는 **2001년 출시 후 2014년까지 Microsoft 공식 지원이 계속됐다.** 13년. 소프트웨어 역사에서 유례없는 수명이었다.

왜 이렇게 오래 살아남았나? Vista가 실패했기 때문이다. 많은 기업과 개인이 Vista로 넘어가지 않고 XP에 머물렀다.

**탄생 이유**: 두 혈통을 통합해 모든 사용자에게 안정적이고 편리한 OS를 제공.
**존재의 가치**: 현재까지도 "이상적인 Windows"의 기준으로 회자된다.
**왜 끝났나**: 보안 지원 종료(2014). 그러나 일부 ATM, 산업용 기기에는 2020년대까지도 살아남았다.

---

## 9. Windows Vista — 야망이 만든 재앙 (2007)

**Vista는 너무 많은 것을 한 번에 바꾸려다 실패한 사례다.**

### 무엇을 바꾸려 했나

Vista의 개발 코드명은 "Longhorn". 야심찼다.

```
Vista가 새로 만든 것들 (동시에):
  ├ Aero 인터페이스: 유리 효과, 3D 전환 → GPU 가속 필요
  ├ 새 드라이버 모델 (WDDM): 그래픽 드라이버 완전 재작성
  ├ 새 오디오 스택: 오디오 드라이버 재작성
  ├ 새 네트워크 스택: TCP/IP 스택 재작성
  ├ UAC (User Account Control): 권한 상승 시마다 확인
  ├ DRM (디지털 저작권 관리): HD DVD/Blu-ray 보호
  ├ WPF (Windows Presentation Foundation): 새 UI 프레임워크
  ├ WCF, WF: 새 서비스·워크플로 프레임워크
  └ BitLocker: 전체 디스크 암호화
```

이 모든 것을 동시에 바꿨다. 문제는 **하드웨어와 드라이버 생태계가 준비되지 않았다**는 것이다.

### Vista의 실패 이유들

**하드웨어 요구사항이 시대를 앞섰다**

Aero 인터페이스는 GPU 가속을 필요로 했다. 2007년 평균 PC의 GPU는 Vista의 Aero를 제대로 돌리기에 부족했다. "Vista Capable" 스티커를 붙인 PC들이 Aero를 못 돌리는 사태가 벌어져 집단 소송까지 갔다.

**드라이버 호환성 붕괴**

새 드라이버 모델(WDDM)은 XP용 드라이버와 호환되지 않았다. 프린터, 스캐너, 그래픽 카드 — 수많은 주변기기가 Vista에서 동작하지 않았다. 제조사들이 Vista용 드라이버를 내놓는 데 시간이 걸렸다.

**UAC의 공포**

User Account Control은 보안을 위해 만들어졌다. 원리는 좋았다. 실행은 최악이었다.

```
Vista UAC의 실제 경험:
  파일 하나 복사하려 해도 → "계속하시겠습니까?" 팝업
  설정 변경 → 팝업
  프로그램 설치 → 팝업
  프로그램 실행 → 팝업
  ...
```

Mac에서 조용히 나온 미소 짓는 Genius와 짜증난 Vista PC를 비교한 Apple의 "Get a Mac" 광고 시리즈가 이 경험을 풍자해 대히트했다.

**성능**

같은 하드웨어에서 XP보다 Vista가 느렸다. 메모리도 더 많이 먹었다. "내 컴퓨터가 느려졌다"는 불만이 쏟아졌다.

**기업 시장의 거부**

많은 기업이 Vista를 아예 채택하지 않았다. 호환성 문제와 교육 비용, 새 하드웨어 구매 부담. "XP 다운그레이드 권리"를 행사해 Vista PC를 사서 XP로 재설치하는 경우가 흔했다.

**탄생 이유**: XP의 오래된 기반 위에 현대적인 UI, 보안, 멀티미디어 기능을 얹는 대규모 혁신.
**존재의 가치**: 기술적으로 Vista에서 도입된 것들 — WDDM, 새 오디오 스택, WPF, UAC의 개념 — 은 Windows의 미래 기반이 됐다. 씨앗은 맞았다. 열매를 너무 일찍 땄다.
**왜 끝났나**: Windows 7이 나오면서 시장에서 신속히 외면받았다.

## 10. Windows 7 — Vista를 완성하다 (2009)

"Windows 7은 Vista SP3이다"라는 말이 있다. 반은 맞고 반은 틀리다.

### Vista에서 달라진 것

기술적 기반은 Vista와 같다. 커널 버전도 NT 6.1(Vista가 6.0)이다. 그러나 경험은 완전히 달랐다.

**UAC 조정**: 팝업 빈도를 낮췄다. 사용자가 직접 조정할 수 있게 했다.

**성능**: Vista보다 적은 RAM, 빠른 부팅. 같은 하드웨어에서 눈에 띄게 빨랐다.

**하드웨어 생태계의 성숙**: Vista 출시 후 2년간 제조사들이 드라이버를 정비했다. 7이 나왔을 때는 대부분 제대로 동작했다.

**Taskbar 개편**: 애플리케이션 아이콘을 Taskbar에 고정. Aero Peek(창 미리보기). 자연스러운 UX.

**Windows XP Mode**: 가상화로 XP 환경을 7 안에서 실행. 기업의 레거시 애플리케이션 우려를 잠재웠다.

**Libraries**: 파일을 물리적 위치 관계없이 묶어서 보는 가상 폴더 개념.

7은 출시 직후부터 시장이 환호했다. XP에서 Vista를 건너뛰었던 기업들이 7으로 대거 이동했다. 2023년까지도 일부 환경에서 사용됐다.

**탄생 이유**: Vista의 개념을 완성하고 경험을 개선.
**존재의 가치**: 안정성과 완성도. "완벽한 Windows"라는 두 번째 평가를 받은 버전.
**왜 끝났나**: 보안 지원 종료(2020). 그러나 기업 환경에서는 훨씬 오래 살아남았다.

## 11. Windows 8 / 8.1 — 또 다른 실패 (2012~2013)

**Vista가 기술적 과잉의 실패였다면, 8은 전략적 오판의 실패였다.**

### 태블릿 전쟁의 공포

2010년 iPad가 출시됐다. 스마트폰이 폭발적으로 성장하고 있었다. Microsoft는 공포에 빠졌다.

"PC가 죽어가고 있다. 터치스크린 시대에 적응하지 못하면 Windows도 죽는다."

이 판단이 Windows 8을 만들었다. 결론부터 말하면: **이 전제 자체가 틀리지 않았지만, 실행이 잘못됐다.**

### 두 개의 UI가 공존하는 혼란

Windows 8의 핵심 변화:

```
Start Screen (Metro/Modern UI):
  터치 최적화, 큰 타일, 전체 화면 앱
  → 키보드+마우스 사용자에게 극도로 불편

Desktop:
  기존 Windows 7과 동일한 UI
  → 하지만 시작 버튼이 없음 (!!)

두 UI 사이를 전환하는 것이 일상이 됨
→ 사용자가 어느 세계에 있는지 혼란
```

**시작 버튼 제거**는 상징적 실수였다. 1995년부터 17년간 있던 버튼이 사라졌다. 많은 사용자가 첫 화면에서 종료 방법을 찾지 못했다. 마우스를 화면 오른쪽 모서리로 가져가야 "Charms Bar"가 나타났다. 발견 불가능한 UI였다.

**Charms Bar**: 오른쪽에서 나타나는 설정·공유·검색 바. 터치에선 자연스러웠지만 마우스로는 실수로 열기 쉬웠다.

**Metro App**: 전체 화면에서만 동작. 창으로 줄일 수 없었다(초기 버전). PC에서 웹브라우저를 전체 화면으로만 쓰는 것을 상상해보라.

### Windows RT — 또 다른 혼란

Surface 태블릿과 함께 **Windows RT**가 출시됐다. ARM 프로세서용 Windows였다. 겉모습은 Windows 8과 같았다. 그러나 기존 x86 데스크탑 프로그램이 하나도 동작하지 않았다. Metro App만 설치 가능했다.

소비자들이 혼란에 빠졌다. "이게 Windows인데 왜 내 프로그램이 안 되지?" Surface RT는 처참하게 실패했다.

### Windows 8.1 (2013) — 부분적 수정

시작 버튼이 돌아왔다. 눌러도 Start Screen으로 가는 버튼이었지만, 심리적 위안이었다. 앱 창모드 지원. 그러나 근본적인 두 UI 혼재 문제는 해결되지 않았다.

기업 시장은 8을 거의 채택하지 않았다. 7에서 바로 10으로 넘어갔다.

**탄생 이유**: 모바일·터치 시대에 대응한 UI 패러다임 전환.
**존재의 가치**: UWP(Universal Windows Platform)의 씨앗. Windows 스토어 생태계의 시작. ARM 지원의 시도.
**왜 끝났나**: 10이 나오면서. 기업은 처음부터 스킵했다.

## 12. 퐁당퐁당의 법칙 — 왜 반복됐나

이제 패턴을 분석할 수 있다.

```
3.1  ──(성공)── 95  ──(성공)── 98 ──(보통)── ME ──(실패)──
XP   ──(성공)── Vista ──(실패)── 7 ──(성공)── 8 ──(실패)──
10   ──(성공)──
```

실패한 버전들 — ME, Vista, 8 — 에는 공통점이 있다.

### 패턴 1: 기반이 준비되지 않았는데 출시

**ME**: NT 커널이 준비돼 있었는데 굳이 Win9x로 만들었다. 소비자용 NT를 만드는 데 시간이 필요했다는 이유였지만, 결과는 구식 기반으로 만든 불안정한 OS였다.

**Vista**: 새 드라이버 모델, 새 그래픽 스택, 새 오디오 스택. 모두 옳은 방향이었지만 하드웨어 제조사와 드라이버 생태계가 준비되지 않은 상태에서 출시됐다. 2년 후 Windows 7이 나왔을 때 생태계는 준비돼 있었다.

**Windows 8**: 터치 UI 전환의 방향은 맞았지만, 태블릿과 PC를 같은 UI로 강제하는 것은 너무 앞서갔다. 그리고 기업 시장의 현실(키보드+마우스)을 무시했다.

### 패턴 2: 성공한 버전은 실패한 버전을 수정한 것

**XP = NT 커널로 Win9x 계열을 대체** (ME가 해야 했던 것을 완성)
**Windows 7 = Vista의 개념 + 경험 개선** (Vista가 되었어야 할 것)
**Windows 10 = Windows 8의 전략 + 경험 개선** (8이 되었어야 할 것)

성공한 버전은 혁신적이지 않았다. 이전 버전의 실수를 고쳤다. **혁신은 실패했고, 완성이 성공했다.**

### 패턴 3: Microsoft의 조직적 압박

Microsoft는 2~3년 주기로 새 버전을 출시해야 하는 압박을 받았다. 라이선스 수익이 버전 업그레이드에 달려있었기 때문이다. 준비되지 않은 기반 위에 새 버전을 출시해야 하는 구조적 문제가 있었다.

Vista는 5년의 개발 기간을 거쳤는데도 실패했다. 오히려 개발 기간이 길어지면서 "이것도 넣자, 저것도 넣자"는 욕심이 커졌고, 결국 완성도보다 범위가 문제였다.

## 13. Windows 10 — 퐁당퐁당이 끝난 이유 (2015)

Windows 10은 여러 면에서 패러다임을 바꿨다.

### "마지막 버전"의 선언

Microsoft는 처음에 Windows 10을 "Windows의 마지막 버전"이라고 했다. 더 이상 숫자가 올라가는 버전 업그레이드가 없고, Windows 10이 서비스처럼 계속 업데이트된다는 의미였다. 실제로 2021년 Windows 11이 나오면서 선언은 철회됐지만, 이 전략이 만든 변화는 컸다.

**무료 업그레이드**: Windows 7/8.1 사용자에게 1년간 무료 업그레이드를 제공했다. 시장 파편화(여러 버전이 혼재)를 해결하기 위해서였다. Windows 10의 채택 속도가 이전 버전보다 훨씬 빨랐다.

### 8의 전략을 유지하되 경험을 고쳤다

- **시작 메뉴 복귀**: 7의 시작 메뉴 + 8의 라이브 타일을 결합
- **데스크탑 우선**: 키보드+마우스 환경이 기본. 태블릿 모드는 선택
- **Continuum**: PC에서 태블릿으로, 태블릿에서 PC로 자동 전환
- **Cortana**: 음성 비서 내장
- **Microsoft Edge**: IE의 후계자 (나중에 Chromium 기반으로 재작성)
- **Windows Hello**: 얼굴/지문 인식 로그인
- **WSL (Windows Subsystem for Linux)**: Linux를 Windows 안에서 실행 (개발자 커뮤니티에서 대히트)
- **DirectX 12**: 차세대 그래픽 API
- **가상 데스크탑**: Mac처럼 여러 바탕화면 전환

### 서비스로서의 Windows

10부터 Windows는 6개월마다 기능 업데이트를 받는 "서비스" 모델로 전환됐다. 큰 버전 전환 없이 점진적으로 기능이 추가됐다. 기업 입장에서는 예측하기 어렵다는 불만도 있었지만, 과거처럼 불완전한 버전을 큰 패키지로 출시하는 리스크가 줄었다.

**퐁당퐁당이 끝난 이유**: 큰 버전 전환 대신 점진적 업데이트. 실패할 만한 빅뱅 출시가 없어졌다. 사용자는 문제가 있으면 다음 달 업데이트에서 수정된다는 것을 안다.

## 14. Windows 11 — 새 시작인가 또 다른 8인가 (2021)

2021년 깜짝 발표됐다. "마지막 버전"이라던 10 이후 6년 만이었다.

**새로운 것들:**
- 센터 정렬된 시작 메뉴 (Mac 느낌)
- Snap Layout: 창 배치 보조
- Android 앱 실행 (WSA, Windows Subsystem for Android — 이후 종료)
- Teams 통합
- 새 설정 UI

**논란:**
- **TPM 2.0 요구**: 2017년 이전 출시된 많은 PC가 공식 지원에서 제외됐다. "내 멀쩡한 컴퓨터가 Windows 11 불가"
- **인터넷 계정 강제**: 설치 시 Microsoft 계정 로그인 강제(우회 방법 있음)
- **광고**: 시작 메뉴에 권장 앱 광고 표시

**퐁당퐁당 재발 여부**: 11은 8만큼의 실패는 아니었지만, 기업 채택이 예상보다 느렸다. 10의 ESU(연장 보안 업데이트) 종료(2025)를 앞두고 기업들이 마지못해 이동하는 양상이었다.

## 15. 소프트웨어 스택의 진화

Windows의 역사는 커널만이 아니다. 그 위의 소프트웨어 스택도 함께 진화했다.

```
개발 플랫폼의 진화:

## 정리

```
두 혈통:
  Win9x (DOS 기반): 3.1 → 95 → 98 → ME → (종료)
  NT 커널:          NT 3.1 → 4.0 → 2000 → XP → Vista → 7 → 8 → 10 → 11

XP에서 통합 (2001) → Win9x 혈통의 종언, NT 기반으로 단일화

퐁당퐁당의 법칙:
  실패 = 생태계 준비 전 출시 (Vista), 구식 기반 고집 (ME),
         전략적 오판 (8)
  성공 = 실패한 버전의 완성 (XP, 7, 10)
  혁신이 실패하고, 완성이 성공했다

퐁당퐁당의 종료:
  10부터 점진적 업데이트 모델 → 빅뱅 출시 없음
  큰 리스크 없이 조금씩 변화 → 단기 대실패 없음

NT 커널이 만든 현재:
  선점형 멀티태스킹, 메모리 보호, NTFS, HAL —
  Dave Cutler가 1988년 설계한 아이디어들이
  2026년에도 Windows의 심장에서 돌아가고 있다
