"메모리가 부족하다"는 말은 상황에 따라 RAM을 뜻하기도 하고, 스토리지를 뜻하기도 하고, GPU 메모리를 뜻하기도 한다. 메모리는 단일 개념이 아니라 속도와 역할에 따라 계층을 이루는 여러 기술의 집합이다. 레지스터부터 HDD까지, 전부 "데이터를 저장한다"는 공통점이 있지만 속도와 목적이 수십만 배씩 차이 난다.

---

## 메모리 계층 구조

메모리는 **빠를수록 비싸고 작다**는 원칙 위에 계층을 이룬다.

```
빠름 / 비쌈 / 작음
  │
  ├─ 레지스터        (CPU 내부, ps 단위, 수십~수백 바이트)
  ├─ L1 캐시         (CPU 내부, ~1ns, 수십 KB)
  ├─ L2 캐시         (CPU 내부/근접, ~5ns, 수백 KB ~ 수 MB)
  ├─ L3 캐시         (CPU 외부 공유, ~20ns, 수십 MB)
  ├─ 메인 메모리(RAM) (~70ns, GB 단위)
  ├─ NVMe SSD        (~100μs, TB 단위)
  ├─ SATA SSD        (~200μs, TB 단위)
  └─ HDD             (~10ms, TB 단위)
느림 / 쌈 / 큼
```

OS는 이 계층을 투명하게 관리한다. 프로그래머가 "L2 캐시에 저장해라"고 명시하지 않아도 CPU가 자주 쓰는 데이터를 자동으로 캐시로 올린다.

---

## 휘발성 vs 비휘발성

메모리를 나누는 첫 번째 기준이다.

| 구분 | 전원 끊기면 | 대표 |
|---|---|---|
| **휘발성 (Volatile)** | 데이터 사라짐 | 레지스터, 캐시, RAM |
| **비휘발성 (Non-volatile)** | 데이터 유지 | ROM, Flash, SSD, HDD |

RAM이 빠른 이유 중 하나가 이것이다. 데이터를 전기 신호로 유지하기 때문에 읽기/쓰기가 빠르지만, 전원이 끊기면 사라진다.

---

## 1. 레지스터 (Register)

CPU 안에 있는 가장 빠른 저장소다. 속도는 빠르지만 수십～수백 바이트 수준으로 극히 작다. 현재 연산 중인 값, 메모리 주소, 상태 플래그 등을 임시로 담는다.

```
x86-64: RAX, RBX, RCX, RDX, RSP, RBP, RSI, RDI + R8~R15 (16개 범용)
ARM64:  X0~X30 + SP, PC, XZR (31개 범용)
```

프로그래머가 직접 접근하는 경우는 어셈블리나 매우 저수준 최적화에서만이다. 컴파일러가 자동으로 레지스터 할당을 최적화한다.

---

## 2. 캐시 (Cache)

CPU와 RAM 속도 차이를 메우는 완충 메모리다. CPU가 어떤 데이터에 접근할 때 먼저 L1 → L2 → L3 순으로 찾고, 없으면 RAM에서 가져온다(캐시 미스).

| 레벨 | 위치 | 크기 | 레이턴시 | 특징 |
|---|---|---|---|---|
| L1-I / L1-D | 코어 내부 | 32～64 KB | ～1ns | 명령어/데이터 분리 |
| L2 | 코어 내부 또는 코어 전용 | 256 KB ～ 4 MB | ～5ns | 코어 전용 |
| L3 | 코어 공유 (LLC) | 4 MB ～ 128 MB | ～20ns | 모든 코어가 공유 |

SRAM(Static RAM)으로 구현된다. DRAM보다 6～8배 비싸고, 같은 면적에 더 적게 집적되므로 용량을 늘리는 데 한계가 있다.

**캐시가 성능을 좌우하는 이유:**
행렬 연산처럼 규칙적으로 메모리에 접근하는 코드는 캐시 히트율이 높아 매우 빠르다. 반면 포인터가 난무하는 자료구조(링크드 리스트 등)는 메모리 접근 패턴이 불규칙해 캐시 미스가 많이 발생한다. 같은 데이터양이라도 배열이 링크드 리스트보다 빠른 이유가 캐시다.

## 3. 메인 메모리 — RAM

### DRAM (Dynamic RAM)

메인 메모리의 기본 단위다. 데이터를 커패시터(축전기)에 전하로 저장한다. 전하가 자연 방전되므로 주기적으로 재충전(Refresh)이 필요하다. 이 "동적(Dynamic)" 새로고침이 이름의 유래다.

SRAM보다 느리지만 같은 면적에 훨씬 많은 용량을 집적할 수 있어 대용량 메인 메모리에 쓰인다.

### SRAM (Static RAM)

플립플롭 회로로 데이터를 유지한다. Refresh가 필요 없어 DRAM보다 빠르다. 하지만 셀 하나에 트랜지스터가 6개 필요해 같은 면적에 DRAM의 1/4～1/6밖에 집적되지 않는다. 비싸고 작다 → 캐시에 사용.

### SDRAM과 DDR의 역사

| 규격 | 연도 | 최대 대역폭 | 전압 |
|---|---|---|---|
| SDRAM | 1993 | ～800 MB/s | 3.3V |
| DDR1 | 2000 | ～3.2 GB/s | 2.5V |
| DDR2 | 2003 | ～6.4 GB/s | 1.8V |
| DDR3 | 2007 | ～17 GB/s | 1.5V |
| DDR4 | 2014 | ～25 GB/s | 1.2V |
| DDR5 | 2020 | ～51 GB/s+ | 1.1V |

DDR(Double Data Rate)은 클럭의 상승 엣지와 하강 엣지 양쪽에서 데이터를 전송한다. 클럭 주파수는 같아도 데이터 전송량이 두 배다. DDR5는 뱅크 구조를 개편하고 전력 관리도 모듈 자체에서 담당한다.

**DDR 세대 간 호환 불가:** DDR4 슬롯에 DDR5 모듈이 물리적으로 들어가지 않는다(노치 위치가 다르다). 호환이 필요하면 동일 세대를 써야 한다.

### LPDDR (Low Power DDR)

모바일 기기와 울트라씬 노트북용으로 설계된 DDR. 전력 소비를 낮추기 위해 다양한 저전력 상태를 지원한다.

| 규격 | 기반 | 전압 | 주 사용처 |
|---|---|---|---|
| LPDDR4/4X | DDR4 | 1.1V | 스마트폰, 태블릿 |
| LPDDR5/5X | DDR5 | 1.05V | 최신 플래그십 스마트폰 |
| LPDDR5X | DDR5 개선 | 1.01V | Snapdragon 8 Gen 3, Apple M 시리즈 |

Apple의 M 시리즈 칩은 LPDDR를 SoC에 통합(온패키지)해서 표준 DIMM보다 훨씬 낮은 레이턴시를 달성한다. 단, 사용자가 메모리를 교체할 수 없다.

### ECC RAM (Error-Correcting Code)

서버와 워크스테이션용. 메모리 셀에서 발생하는 단일 비트 오류를 자동으로 감지하고 수정한다. 오류 정정용 추가 비트를 저장하기 위해 일반 64비트 데이터 폭 외에 8비트가 추가된다(72비트 총 폭).

우주방사선(Cosmic Ray)이나 열에 의한 비트 플립이 금융, 의료, 과학 계산에서는 치명적이다. ECC는 이를 막는 장치다. AMD EPYC, Intel Xeon 기반 서버가 ECC를 지원하며, 일반 데스크탑 CPU는 대부분 ECC를 지원하지 않는다(AMD Ryzen 일부 지원, Intel Core는 미지원).

## 4. HBM (High Bandwidth Memory)

GPU, AI 가속기, HPC(고성능 컴퓨팅)용 고대역폭 메모리다. 일반 GDDR보다 훨씬 넓은 데이터 버스를 사용한다.

```
GDDR6:  256비트 버스, ~672 GB/s (RTX 4090)
HBM3e:  1024비트 버스(스택당), 5 TB/s+ (NVIDIA H200)
```

HBM은 DRAM 다이를 수직으로 쌓고(3D 스태킹) 인터포저(중간 기판) 위에 GPU와 나란히 배치해서 거리를 최소화한다. 이 구조가 넓은 버스폭과 낮은 레이턴시를 가능하게 한다.

| 세대 | 대역폭/스택 | 주 사용처 |
|---|---|---|
| HBM2 | ～256 GB/s | AMD Radeon VII, Tesla V100 |
| HBM2e | ～461 GB/s | A100 |
| HBM3 | ～819 GB/s | H100 |
| HBM3e | ～1.2 TB/s | H200, MI300X |

AI 붐으로 HBM 수요가 폭발하면서 SK하이닉스, 삼성, Micron이 생산 경쟁 중이다. HBM은 현재 AI 반도체 공급망의 핵심 병목이다.

## 5. 비휘발성 메모리

### ROM (Read-Only Memory)

공장에서 데이터를 구워 넣고 이후 수정 불가. 현재는 순수 ROM은 거의 없고 EPROM/EEPROM/Flash로 대체됐다.

- **EPROM**: UV 자외선으로 지우고 재기록 가능
- **EEPROM**: 전기 신호로 바이트 단위 수정 가능. 설정값 저장에 사용

### Flash Memory

현대 비휘발성 스토리지의 핵심이다. 전기 신호로 셀에 전자를 주입/방출해 데이터를 저장한다. 블록 단위로 지우고 재기록한다.

두 가지 구조가 있다.

**NOR Flash**
- 셀이 병렬 연결 → 임의 주소를 직접 읽기 가능(Random Access)
- 읽기가 빠르고 바이트 단위 실행 가능
- 쓰기/지우기가 느리고 용량이 작음
- 사용처: 펌웨어(BIOS/UEFI), 마이크로컨트롤러 코드 저장

**NAND Flash**
- 셀이 직렬 연결 → 페이지/블록 단위 접근
- 쓰기와 대용량 집적에 유리
- 사용처: SSD, eMMC, UFS, USB 드라이브, SD 카드

### NAND 셀 세대 — 비트 수에 따른 분류

셀 하나에 몇 비트를 저장하느냐에 따라 속도, 수명, 가격이 달라진다.

| 종류 | 비트/셀 | P/E 사이클 | 속도 | 가격 | 용도 |
|---|---|---|---|---|---|
| SLC | 1 | ～100,000 | 가장 빠름 | 매우 비쌈 | 산업용, 엔터프라이즈 SSD |
| MLC | 2 | ～10,000 | 빠름 | 비쌈 | 고성능 SSD |
| TLC | 3 | ～3,000 | 중간 | 보통 | 소비자 SSD 주류 |
| QLC | 4 | ～1,000 | 느림 | 쌈 | 대용량 저가 SSD |
| PLC | 5 | ～300 | 가장 느림 | 가장 쌈 | 초대용량 (실험적) |

P/E 사이클은 쓰고 지우기를 반복할 수 있는 횟수다. QLC SSD가 TLC보다 수명이 짧은 이유다. TBW(Total Bytes Written) 스펙이 낮은 SSD는 대체로 QLC 기반이다.

**3D NAND (V-NAND)**
셀을 평면이 아니라 수직으로 쌓아 용량을 늘리는 기술. 삼성이 V-NAND로 명명했다. 128단, 176단, 232단까지 적층 단수가 증가하면서 가격 대비 용량이 향상됐다.

### eMMC vs UFS

모바일 기기의 내장 스토리지 규격이다.

| 규격 | 인터페이스 | 최대 속도 | 특징 |
|---|---|---|---|
| eMMC 5.1 | 병렬 | ～400 MB/s | 저가 스마트폰, 태블릿 |
| UFS 3.1 | 직렬 (양방향 동시) | ～2,100 MB/s | 중고급 스마트폰 |
| UFS 4.0 | 직렬 | ～4,200 MB/s | 플래그십 (Snapdragon 8 Gen 3) |

eMMC는 읽기와 쓰기를 동시에 할 수 없다(반이중). UFS는 읽기와 쓰기를 동시에 할 수 있어(전이중) 전반적인 체감 속도가 빠르다.

## 6. VRAM (Video RAM)

GPU에 탑재된 전용 메모리다. 프레임 버퍼(화면에 그릴 픽셀 데이터), 텍스처, 셰이더 데이터를 저장한다. GPU는 수천 개의 코어가 동시에 메모리에 접근하므로 일반 RAM보다 훨씬 높은 대역폭이 필요하다.

| 규격 | 대역폭 | 사용 GPU |
|---|---|---|
| GDDR5 | ～336 GB/s | GTX 900 시리즈 |
| GDDR6 | ～672 GB/s | RTX 3090 |
| GDDR6X | ～1,008 GB/s | RTX 4090 |
| HBM3e | ～5 TB/s | H200, MI300X |

소비자 GPU는 GDDR 계열, AI/HPC 가속기는 HBM 계열을 쓴다. VRAM 용량이 AI 추론에서 핵심 병목인 이유는 모델 파라미터 전체가 VRAM에 올라와야 하기 때문이다. 70B 파라미터 모델(FP16)은 약 140GB VRAM이 필요하다.

## 7. 새로운 종류들

### Optane / 3D XPoint (단종)

Intel과 Micron이 공동 개발한 비휘발성 메모리. DRAM과 NAND의 중간 특성을 목표로 했다. NAND보다 훨씬 빠르고 수명이 길었지만, NAND/DRAM 대비 가격 경쟁력이 없어 Intel이 2022～2023년 사업을 철수했다.

### MRAM (Magnetoresistive RAM)

자기 터널 접합으로 데이터를 저장하는 비휘발성 메모리. 읽기/쓰기 속도가 DRAM에 가깝고, 무한에 가까운 쓰기 수명을 가진다. 현재는 소용량 임베디드 분야에서 사용 중이며 범용 메모리로 확장 중이다.

### CXL (Compute Express Link) 메모리

PCIe 기반 메모리 확장 기술. 서버에서 CPU가 마치 로컬 메모리처럼 외부 메모리 풀에 접근할 수 있게 한다. 메모리 용량 제약을 넘기 위한 차세대 데이터센터 기술이다.

## 주의사항 및 한계

**용량 ≠ 성능**
RAM이 32GB라도 DDR4-2400이 DDR5-6400보다 느리다. 용량과 속도는 다른 축이다.

**듀얼채널 / 쿼드채널**
같은 용량이라도 메모리 슬롯 구성에 따라 대역폭이 달라진다. 8GB 1개보다 4GB 2개(듀얼채널)가 이론상 2배 대역폭이다. 게임보다 메모리 대역폭에 민감한 작업(영상 편집, ML)에서 차이가 크다.

**NAND 수명과 SSD 선택**
소비자 SSD의 TLC는 일반 사용에 충분하지만, 지속적으로 대량 쓰기가 필요한 서버나 로그 저장 용도엔 MLC/SLC 기반 엔터프라이즈 SSD를 써야 한다. TBW 수치를 확인하는 습관이 필요하다.

**메모리 규격 혼용 불가**
DDR4와 DDR5는 핀 수도 다르고 노치 위치도 다르다. 메인보드가 지원하는 규격에 맞는 메모리를 선택해야 한다. LPDDR은 납땜되어 있어 교체가 불가능하다(Apple Silicon, 일부 울트라씬 노트북).

**VRAM은 RAM과 다르다**
Windows의 작업 관리자에서 GPU 메모리를 "RAM"이라고 표시하는 경우가 있어 혼동이 생긴다. VRAM은 GPU 전용이고 CPU 프로세스가 직접 접근할 수 없다(명시적 복사 필요).
