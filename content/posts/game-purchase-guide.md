게임을 산다는 행위는 CD 한 장을 사던 시절과 지금이 완전히 다르다. 지금 지불하는 돈은 콘텐츠 값이 아니라 서비스 이용권에 가깝고, 그 서비스는 언제든 회사의 판단이나 규제로 끊길 수 있다. 이 변화를 이해하지 못하면 "세일할 때 사면 된다"는 단순한 원칙조차 제대로 적용할 수 없다. 요즘 생태계가 어떻게 바뀌었는지부터, 개인·단체 단위의 실전 구매 기준까지 정리한다.

---

## 1. 요즘 게임 생태계 — 패키지에서 상시 서비스로

### 지금 사는 건 콘텐츠가 아니라 서비스다

과거의 패키지 게임은 CD 한 장을 사면 그걸로 거래가 끝났다. 지금은 다르다. 구매 시점에 받는 것은 완성품이 아니라 **서버 접속권, 매칭 시스템, 주기적인 버그 픽스와 밸런스 패치를 포함한 서비스 이용권**이다.

```
과거: 돈을 낸다 → 완성된 소프트웨어를 받는다 → 거래 종료
현재: 돈을 낸다 → 서버·매칭·패치·밸런싱이 포함된 서비스를 받는다 → 거래는 계속 이어짐
```

이 관점에서 보면 게임 가격은 "물건 값"이 아니라 "지속되는 서비스에 대한 선불 결제"에 가깝다.

### 가격이 우상향할 수밖에 없는 이유

환율 변동과 개발 비용 상승(그래픽 품질·인건비·마케팅비 증가)은 구조적으로 게임 가격을 계속 밀어올린다. 이건 일시적 현상이 아니라 산업 전체의 방향이므로, 개인이 "예전엔 안 그랬는데"라며 저항해봐야 흐름을 바꿀 수 없다. 대신 필요한 건 **지출에 대한 개인 단위의 규율**이다 — 무엇을, 언제, 얼마나 살지에 대한 스스로의 기준을 세우는 것.

### 스팀 평가로 여론을 읽는 법

스팀 평가는 단순한 별점이 아니라 유저가 회사에 압력을 넣는 사실상의 시위 수단이다. 다만 숫자를 그대로 믿으면 안 되고, 아래 기준으로 걸러 읽어야 한다.

| 확인 항목 | 이유 |
|---|---|
| **전체 평가 vs 최근 평가 분리** | 전체는 좋아도 최근 평가만 급락했다면 최근 업데이트·서버 상태에 문제가 생겼다는 신호 |
| **리뷰 폭탄 vs 실제 결함 구분** | 운영 이슈(정책 변경, 회사 태도)에 대한 감정적 항의인지, 게임성 자체의 결함인지는 리뷰 본문을 몇 개만 훑어도 대체로 구분됨 |
| **리뷰 개수(표본 크기)** | 평가 비율이 좋아도 표본이 100개 미만이면 신뢰도가 낮음 |

그리고 가장 위험한 신호는 **평가가 오래도록 변하지 않는 것**이다. 이는 개발사·퍼블리셔·유통사가 그 게임을 사실상 유기했다는 뜻으로 읽을 수 있다 — 좋은 쪽으로든 나쁜 쪽으로든 아무 변화가 없다는 건 아무도 그 게임을 더 이상 신경 쓰지 않는다는 의미다.

## 2. 소유가 아니라 대여 — 디지털 구매의 숨은 전제

여기서 반드시 짚어야 할 전제가 하나 있다. **디지털 배포 시대의 "구매"는 소유가 아니라 접근권 대여에 훨씬 가깝다.** 이 주제는 별도로 깊게 다룰 만큼 이유가 많아 여기서는 짧게만 정리한다.

```
구매했다고 믿는 것 : 게임 파일에 대한 영구 소유권
실제로 받은 것    : 회사의 서버·정책·판단이 유지되는 동안의 접근 권한
```

이 전제가 현실에서 드러나는 대표적인 세 가지 방식이 있다.

- **판매 부진으로 인한 스토어 하차**: 수익성이 없다고 판단된 타이틀이 별다른 사전 고지 없이 스토어에서 내려가는 경우
- **규제로 인한 강제 하차**: 국내 기준 청소년 이용불가 등급 게임이 주기적으로 등급 재심사나 규제 변화로 스토어에서 벤 당하는 경우
- **게임사 자체 판단에 의한 회수**: 유비소프트가 온라인 필수 레이싱 게임의 서버를 종료하며 싱글 콘텐츠까지 포함해 전체를 실행 불가 상태로 만든 사례가 대표적이다. 이미 돈을 낸 유저의 접근권까지 소급으로 막았다는 점에서 업계에 큰 반발을 일으켰다("Stop Killing Games" 운동으로 이어짐)

세 경우 모두 결론은 같다 — **돈을 냈다는 사실이 영구적인 접근을 보장하지 않는다.** 이 인식이 있어야 뒤에 나올 구매 기준들(특히 "정말 좋아하는 게임만 산다", "게임 외적인 것은 사지 않는다")이 왜 필요한지가 명확해진다. 자세한 사례와 구조적 원인은 다음 글에서 따로 다룬다.

## 3. 구매 판단 기준 — 개인 단위

```
원칙 1: 세일할 때만 산다
  → 정가 구매는 원칙적으로 하지 않는다
  → 보조 장치: 스팀 환불 정책(2시간 미만 플레이 + 14일 이내)을 안전판으로 활용해,
    애매하면 일단 사서 확인 후 환불하는 것도 합리적 전략이다

원칙 2: 평가가 부정적이지 않을 때 산다
  → 앞서 다룬 대로 "최근 평가"와 "전체 평가"를 분리해서 볼 것
  → DLC·시즌패스는 본편과 별개로 판단할 것 — 본편 쪼개팔기 논란 때문에
    본편 평가와 DLC 평가가 따로 노는 경우가 흔하다

원칙 3: 출시 직후 구매는 지양한다
  → 최근 게임은 출시 초반 최적화·버그 이슈가 흔하다
  → 세일 타이밍과 별개로 "패치가 몇 차례 쌓인 뒤"를 기다리는 편이 안전하다

원칙 4: 얼리 액세스는 별도로 취급한다
  → 완성된 상품을 사는 게 아니라 미완성 개발 과정에 투자하는 것에 가깝다
  → 2번에서 다룬 "대여" 리스크가 가장 높은 카테고리 — 개발 중단 가능성을 항상 열어둘 것

원칙 5: 게임 외적인 것은 사지 않는다
  → 아트북, 사운드트랙 등은 별도 앱으로 언제든 들을 수 있는 게 아닌 이상 실효 가치가 낮다

원칙 6: 정말 좋아하는 게임일 때는 산다
  → 인생에서 손에 꼽을 만큼 좋아서 소장 자체에 의미가 있는 경우
  → 이 조건은 웬만해서는 발동되지 않는다는 전제를 스스로 알고 있어야 한다

원칙 7: 지금 하고 있는 게임이 없을 때 산다
  → 이 조건도 현실적으로 거의 발동되지 않는다
  → 오히려 중요한 건 이 조건이 발동 안 되는데도 세일마다 계속 사들이는
    소비 패턴 자체를 스스로 자각하는 것 — 이미 쌓인 백로그를 외면한 채
    "일단 사두면 언젠가 하겠지"는 대부분 실현되지 않는다

보조 기준: 짧게 소비하고 지울 게임은 구독 서비스(Xbox Game Pass, PS Plus 등)로 대체할 것
  → 구독은 "구매"가 아니라 "대여"이므로, 소장 의미가 없는 게임까지 사서
    쌓아두는 것보다 합리적이다. 오래 소장하고 싶은 게임만 실제 구매로 남긴다
```

## 4. 구매 판단 기준 — 단체 단위

```
원칙 1: 같이 하자는 사람이 있을 때
  → 단, 상대가 충분히 친하고 그 게임에 실제로 열정이 있어야 성립한다
  → 한쪽만 의욕이 있으면 금방 흐지부지된다

원칙 2: 파티형 게임으로 접대할 수 있다는 판단이 들 때
  → 단, 파티를 자주 열 만큼 상황이 되는 경우에나 해당한다 — 보통은 그럴 일이 잘 없다

보조 기준 1: 패밀리 라이브러리 공유 우선 확인
  → 스팀 Family Sharing처럼 계정 공유가 가능한 관계라면
    굳이 각자 살 필요 없이 공유로 해결되는 경우가 있다

보조 기준 2: 멀티플레이 게임은 선구매 의향 확인이 먼저
  → 한 명만 사고 나머지가 안 사면 애초에 의미가 없으므로,
    "다 같이 살 의향이 있는지"를 구매 전에 반드시 확인한다
```

## 정리

```
생태계 변화 → 패키지 구매에서 상시 서비스(서버·매칭·패치)로. 가격은 환율·개발비로 구조적 우상향
평가 읽는 법 → 전체/최근 평가 분리, 리뷰폭탄과 실제결함 구분, 표본 크기 확인, 평가 정체=방치 신호
소유의 전제  → 디지털 구매는 소유가 아니라 대여에 가까움(하차·규제·회사 판단으로 접근권 소급 상실 가능)
개인 구매 기준→ 세일+환불정책 활용, 평가+DLC 분리 판단, 출시 직후 지양, 얼리액세스 별도 취급,
              게임 외적인 것 배제, 진짜 좋아할 때만, 백로그 자각, 짧게 쓸 건 구독으로 대체
단체 구매 기준→ 상대의 열정 전제, 파티 빈도 현실적 판단, 패밀리 공유 우선 확인, 멀티플레이는 선의향 확인
```

결국 이 모든 기준을 관통하는 한 가지는 "지금 사는 게 정말 필요한가"를 세일가·평가·백로그·소유권의 불확실성까지 포함해 냉정하게 판단하는 것이다. 게임은 사는 순간의 만족보다, 산 이후에도 계속 접근 가능한지가 훨씬 중요한 시대가 됐다.
