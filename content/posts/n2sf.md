한국 공공기관 보안의 근간이었던 망분리 정책이 한계에 부딪혔다. 그 대안으로 국가정보원이 발표한 것이 **N²SF(National Network Security Framework, 국가 망 보안체계)**다. 제로 트러스트와 비슷해 보이지만, 출발점과 적용 맥락이 다르다.

---

## 1. N²SF란 무엇인가

N²SF는 국가정보원(NIS)이 주도하여 공공기관의 네트워크 보안 체계를 현대화하기 위해 수립한 **국가 표준 보안 프레임워크**다.

기존 공공기관 보안의 핵심은 **망분리**였다. 인터넷망과 업무망을 물리적 또는 논리적으로 완전히 분리하여, 내부 데이터가 인터넷으로 새지 못하도록 막는 구조다.

```
[기존 망분리]

인터넷용 PC ──── 인터넷망 (외부)
                     ↕ 단절 (물리/논리 분리)
업무용 PC   ──── 업무망 (내부)
                  ├ 행정 시스템
                  └ 내부 자료
```

직원은 인터넷을 쓰려면 인터넷용 PC를, 업무를 하려면 별도의 업무용 PC를 써야 했다. 파일을 옮기려면 승인된 매체(망간자료전송시스템 등)를 거쳐야 했다.

N²SF는 이 구조를 **데이터 중심 보안**으로 전환하는 프레임워크다. 네트워크 경계를 물리적으로 나누는 대신, 데이터의 민감도와 사용자·기기의 신뢰 수준에 따라 접근을 동적으로 제어한다.

```
[N²SF]

사용자/기기 신뢰도 평가
  +
데이터 민감도 분류
  ↓
접근 정책 동적 적용
  ↓
허용된 데이터에만, 허용된 방식으로 접근
```

보안 수준을 **3단계**로 분류하여 운용한다.

| 등급 | 대상 데이터 | 접근 통제 수준 |
|---|---|---|
| **1등급** | 비밀·국가 핵심 정보 | 물리적 망분리 유지 (기존 방식) |
| **2등급** | 민감 업무 정보 | 논리적 분리 + 강화된 인증 |
| **3등급** | 일반 업무 정보 | 통합망 + 접근 제어 |

1등급 정보는 여전히 기존 물리 망분리를 유지한다. 핵심은 모든 것을 다 분리하는 게 아니라, **데이터 성격에 맞는 수준의 통제**를 적용한다는 것이다.

---

### 1-1. 제로 트러스트와 뭐가 다른가

표면적으로 비슷해 보이지만 출발점과 성격이 다르다.

| 구분 | 제로 트러스트 | N²SF |
|---|---|---|
| **성격** | 철학 / 아키텍처 패턴 | 국가 표준 / 규제 프레임워크 |
| **출발점** | 민간 기업 (Google BeyondCorp) | 한국 공공기관 보안 정책 |
| **강제성** | 없음 (선택적 도입) | 공공기관 의무 적용 |
| **기준** | 기술 원칙 중심 | 법령·고시·가이드라인 중심 |
| **물리 망분리** | 언급 없음 (네트워크 무관) | 1등급 정보는 물리 망분리 유지 |
| **주관** | NIST(미국), 업계 컨소시엄 | 국가정보원(한국) |
| **적용 대상** | 모든 조직 | 공공기관, 정부기관 우선 |

제로 트러스트는 "어떻게 보안을 생각할 것인가"에 관한 철학이다. N²SF는 "공공기관이 무엇을 준수해야 하는가"를 규정하는 제도다. N²SF가 제로 트러스트의 원칙을 참조하여 설계됐지만, 한국 공공 환경의 법적 요건과 기존 망분리 체계를 반영해 변형된 형태다.

또 다른 차이는 **망분리를 완전히 버리지 않는다**는 점이다. 제로 트러스트는 네트워크 위치 자체를 신뢰의 기준에서 제외한다. N²SF는 가장 민감한 정보에 대해서는 물리적 분리를 유지하면서, 나머지 영역에 동적 접근 제어를 도입하는 **점진적 전환** 방식을 택했다.

## 2. 존재 의의

### 망분리의 한계가 임계점에 도달했다

망분리는 2000년대 공공기관 정보 유출 사고가 반복되면서 강화된 정책이다. 물리적으로 분리하면 유출 경로 자체가 없어진다는 논리였다. 실제로 특정 유형의 공격을 막는 데 효과가 있었다.

그러나 클라우드 전환과 디지털 정부 추진이 본격화되면서 문제가 드러났다.

**클라우드를 쓸 수 없다**
AWS, Azure 같은 퍼블릭 클라우드 서비스는 인터넷 기반이다. 업무망이 인터넷과 단절되어 있으면 클라우드 네이티브 서비스를 활용하기 어렵다. 디지털 정부 혁신을 추진하면서 망분리가 발목을 잡는 상황이 됐다.

**재택근무와 충돌한다**
코로나19 이후 공공기관도 재택근무가 일반화됐다. 집에서 업무망에 접속하려면 VPN을 써야 하는데, 물리적 망분리 원칙과 충돌한다. 임시방편으로 운용하다 보니 보안 허점이 생겼다.

**보안 효과가 기대에 못 미쳤다**
망분리를 해도 내부자에 의한 USB 반출, 승인된 파일 전송 채널을 통한 악성코드 유입은 막기 어려웠다. 분리된 망 안에서 이미 감염된 시스템이 횡적으로 퍼지는 것도 통제하지 못했다. 2014년 한국수력원자력 해킹, 2016년 국방망 침해 사고는 망분리 환경에서도 발생했다.

**생산성 비용이 너무 크다**
PC 두 대, 승인 절차를 거친 파일 이동, 인터넷 검색도 별도 단말에서 — 이 구조가 업무 효율을 크게 떨어뜨린다는 지적이 계속됐다.

### N²SF가 해결하려는 것

N²SF의 목표는 보안 수준을 낮추는 게 아니라, **필요 이상으로 높은 통제를 합리화**하는 것이다. 모든 데이터에 동일한 최고 수준의 통제를 적용하는 대신, 데이터 민감도에 비례한 통제를 적용한다.

```
기존: 모든 업무 데이터 = 최고 등급 통제
N²SF: 데이터 등급 → 맞춤 통제

일반 공문 → 3등급 → 통합망 + 기본 인증
인사정보  → 2등급 → 논리 분리 + MFA
국가기밀  → 1등급 → 물리 망분리
```

이를 통해 클라우드 도입, 재택근무, 업무 효율화를 가능하게 하면서 핵심 보안을 유지하는 것이 N²SF의 설계 의도다.

## 3. 실제 적용 사례

### 중앙행정기관의 단계적 전환

국가정보원의 N²SF 가이드라인에 따라 중앙행정기관부터 시범 적용이 시작됐다. 전환 순서는 데이터 분류부터 시작한다.

```
1. 기관 보유 데이터 전수 조사
   → 민감도 기준으로 1/2/3등급 분류

2. 3등급 업무부터 통합망으로 전환
   → 일반 행정 문서, 공개 정보 처리 업무

3. 2등급 영역에 논리적 분리 + 강화 인증 도입
   → MFA, 기기 인증, 접근 로그 강화

4. 1등급은 기존 물리 망분리 유지
   → 변경 없음
```

### 공공 클라우드 연계

정부24, 민원 포털 같은 대국민 서비스는 클라우드 전환이 진행 중이다. N²SF 체계에서 이 서비스들은 3등급으로 분류되어 통합망에서 운영할 수 있게 됐다. 기존에는 인터넷망과 업무망이 분리되어 있어 두 환경을 연결하는 복잡한 중계 시스템이 필요했다.

### 재택근무 합법화

N²SF 도입과 함께 공공기관 재택근무 보안 가이드라인이 개정됐다. 기기 인증(MDM), MFA, 접속 로그 기록을 갖춘 환경에서는 재택근무 중 업무망 접속이 허용되는 방향으로 전환됐다.

```
기존: 재택 = 업무망 접속 불가 (망분리 원칙 위반)
N²SF: 재택 + 기기 인증 + MFA = 2/3등급 업무 접근 허용
```

### 군·정보기관은 별도 적용

국방부, 국가정보원 등 보안 요건이 가장 높은 기관은 N²SF의 일반 가이드라인과 별도로 자체 보안 체계를 유지한다. 1등급 정보를 다루는 핵심 시스템은 N²SF에서도 물리 망분리를 명시적으로 유지하도록 규정했다.

## 주의점

**데이터 분류가 핵심이자 난관이다**
N²SF의 전제는 데이터를 올바르게 분류하는 것이다. 분류가 잘못되면 저등급으로 내려간 민감 정보가 노출될 수 있다. 수십 년치 공공 데이터를 분류하는 작업 자체가 대규모 프로젝트다.

**기관별 역량 차이**
중앙부처와 지방 소도시 군청의 보안 인력과 예산은 천양지차다. 동일한 프레임워크를 적용하면 역량이 부족한 기관에서 형식적 준수에 그칠 위험이 있다.

**전환 기간 중 보안 공백**
망분리에서 N²SF로 전환하는 과도기에 일시적인 보안 취약 구간이 생길 수 있다. 단계적 전환 계획과 전환 중 모니터링 강화가 중요하다.

## 해외 유사 사례

N²SF처럼 국가 차원에서 공공기관 네트워크 보안 체계를 재정비한 사례는 여러 나라에 있다. 접근 방식은 조금씩 다르지만 방향은 같다 — 경직된 경계 보안에서 데이터·신원 중심의 동적 제어로.

### 미국 — 연방 제로 트러스트 전략 (EO 14028)

2021년 바이든 행정부는 **행정명령 14028**을 통해 연방 정부 기관 전체에 제로 트러스트 아키텍처 도입을 의무화했다. CISA(사이버보안·인프라보안국)가 **Zero Trust Maturity Model**을 발표하며 5단계(신원, 기기, 네트워크, 애플리케이션, 데이터) 성숙도 기준을 제시했다. 2024년까지 기본 수준 달성을 목표로 했으며, NIST SP 800-207이 기술 표준으로 함께 쓰인다. N²SF와 비교하면 물리 망분리 유지 조항 없이 처음부터 제로 트러스트 원칙만을 따른다는 점이 다르다.

### 영국 — NCSC 네트워크 아키텍처 가이드라인

영국 국가사이버보안센터(NCSC)는 정부 기관을 위한 **네트워크 분리 원칙**과 **제로 트러스트 네트워크 아키텍처** 가이드를 별도로 발표했다. 한국의 망분리처럼 물리적 분리를 의무화하지는 않지만, 데이터 민감도에 따라 분리 수준을 달리하는 위험 기반 접근을 권고한다. 핵심 정부 시스템은 **Government Secure Intranet(GSI)**이라는 별도 네트워크를 쓰며, 이는 N²SF 1등급 망분리와 유사한 개념이다.

### 호주 — Information Security Manual (ISM) + Essential Eight

호주 신호국(ASD)이 운영하는 **ISM(정보보안 매뉴얼)**은 정부 기관의 네트워크 보안 기준을 규정한다. 함께 제시하는 **Essential Eight**은 우선 적용해야 할 8가지 보안 통제(애플리케이션 제어, 패치, 매크로 차단, MFA 등)를 정의한다. 별도의 망분리 의무 대신 데이터 분류 체계(PROTECTED, SECRET, TOP SECRET)에 따라 통제 수준을 달리한다. N²SF의 1/2/3등급 분류와 구조가 닮았다.

### 독일 — BSI IT-Grundschutz

독일 연방정보보안청(BSI)의 **IT-Grundschutz(기본 보호)**는 1990년대부터 이어온 공공기관 보안 기준이다. 네트워크 세그멘테이션과 DMZ 구성을 상세히 규정하며, 기관의 보호 수준(기본·표준·높음·매우 높음)에 따라 요건이 달라진다. 오랜 역사만큼 문서가 방대하고 세부적이다. N²SF가 참조한 프레임워크 중 하나로 알려져 있다.

### 일본 — 정부기관 통일 기준군

일본 내각부 사이버보안전략본부(NISC)는 각 중앙 성청에 적용되는 **정부기관 등의 사이버보안 대책을 위한 통일 기준군**을 운용한다. 한국의 망분리와 유사하게 인터넷 접속 단말과 내부망 단말을 분리하는 구성을 오랫동안 의무화했다. 최근 클라우드 전환 가속화에 따라 이 분리 원칙을 완화하는 방향으로 개정이 논의되고 있으며, 방향성은 N²SF와 같다.

---

각국 모두 동일한 딜레마를 안고 있다. **보안을 유지하면서 클라우드와 원격 근무를 수용해야 한다.** 물리적 경계를 고집하면 경직되고, 경계를 허물면 위협이 들어온다. N²SF는 이 딜레마를 한국식으로 풀어낸 사례로, 해외 사례들과 비교해보면 각국이 비슷한 문제를 다른 속도와 방식으로 해결하고 있음을 알 수 있다.

## 정리

N²SF는 한국 공공기관 보안이 20년 만에 패러다임을 바꾸는 시도다. 물리적 분리라는 단순하지만 경직된 방어에서, 데이터 가치에 비례한 유연한 통제로의 전환이다.

제로 트러스트라는 국제 흐름을 참조하되, 한국 공공 환경의 법제도·기존 인프라·보안 요건에 맞게 설계된 국산 프레임워크다. 완성된 해답이라기보다는 전환의 방향을 제시하는 로드맵에 가깝고, 실제 효과는 각 기관의 데이터 분류 품질과 실행력에 달려 있다.
